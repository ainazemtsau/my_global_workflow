## Purpose

Build the first autonomous training/activity process v0 inside the unified Health/Body Operator Project.

The Phase does not make workflow the daily execution surface. Workflow builds, repairs, audits, and evolves the operating process. The operational Project runs daily nutrition, training, logs, decisions, adjustments, and small local process customization.

## Critical Constraint

Nutrition works enough to continue, but the Direction does not yet have an autonomous training/activity process. Without a clear operational boundary, workflow can accidentally become the place where daily execution is tracked.

## Minimum Outcome

Health/Body Operator Project can run training process v0:

```text
minimal training state -> first-week plan -> log inside Operational Project -> adjustment
```

## Validation Signal

Capability validation proves that the operational Project can perform the loop. Daily logs are not stored or managed in workflow.

# BOOTSTRAP — как начать

Три шага от нуля до работающего направления. Ничего не настраивается заранее, кроме одного чата.

## Шаг 1. Создать направление (один чат, play: frame)

Открой чат (ChatGPT Project по адаптеру, или сессию Claude Code/Codex на этом репо — второе проще: ноль пересылок) и вставь:

```
CALL c-001
to: session
direction: <придумай-id, напр. indie-game>
play: frame
goal: |
  Создать направление: <одно предложение о твоей амбиции>.
context: |
  Правила: os/KERNEL.md. Процедура: os/plays/frame.md.
  Схемы файлов: os/schema/direction-files.md.
  Это новое направление, состояния ещё нет.
boundaries: |
  Только CHARTER и корень дерева. Никаких задач и детальных планов.
done_when: |
  CHARTER.md и TREE.md созданы и приняты мной; в NOW.md лежит CALL
  на первый shape.
return: RESULT по os/schema/packets.md
budget: one session
```

Чат проведёт интервью, соберёт хартию, пре-мортем, корень дерева — и закроется RESULT'ом.

## Шаг 2. Применить RESULT (writer)

Если сессия шла в Claude Code/Codex — она сама закоммитит `live/<id>/...`.
Если в ChatGPT — вставь RESULT в сессию-writer на этом репо со словами: «Apply per os/adapters/coding-agent.md, role writer».

## Шаг 3. Дальше — по циклу

Каждый следующий чат начинается одинаково: возьми `next` из NOW.md (или из последнего RESULT) и вставь в новый чат. Цикл: shape → work×N → review → (решение) → shape… Раз в неделю — pulse.

Если что-то разъехалось или непонятно где ты — вставь в чат одну строку: `play: repair, direction: <id>` и дай ему NOW.md.

## Что НЕ нужно делать

- Не настраивай все направления заранее — направление создаётся, когда есть амбиция (gate G8).
- Не пиши правил для направления — правила живут в os/, направление живёт в live/.
- Не редактируй state-файлы руками — через RESULT и writer, иначе появится десинк (его починит repair, но зачем).

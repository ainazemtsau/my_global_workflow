# Play: frame

Purpose: create a new direction's foundation — charter and tree root ONLY. The goal tree itself is built afterwards in a dedicated map session (one entity per chat). Also the entry point for revising an existing charter. The only entry for new directions (gate G8).

Reads: owner's intent; for revision — CHARTER.md.
Writes: CHARTER.md, TREE.md (root node only), NOW.md, LOG.md.

## Steps

1. **Interview** — ask only what is missing, in one batch (≤7 questions): What outcome makes this direction worth years of your life? Success in numbers? Hard constraints (time/money/health)? What does failure look like? Which competencies does this direction need (these become lenses)? Which product repos exist or will exist?
2. **Charter draft** — mission (1–2 sentences), 2–4 measurable success criteria, constraints, lenses, repos. Show it; iterate until the owner explicitly approves it (gate G9).
3. **Pre-mortem** — "this direction failed three years from now": ≥5 distinct failure reasons, each mapped to a mitigation, a kill_by candidate, or an explicit accepted risk. Into the charter; owner approves.
4. **Outside view** — 2–3 reference cases (how attempts like this usually go, base rates) and what that implies for sequencing. One paragraph in the charter.
5. **Root node** — TREE.md gets ONLY the root: the mission as an outcome with done_when. No children (they are map's job). Everything the interview surfaced as possible goals goes into RESULT.captures as candidate outcomes for the map session.
6. **Close** — RESULT with `owner_approved` marks for charter and root (G9); log line; next = CALL for a map session carrying the candidate outcomes.

## Done when

CHARTER.md and the tree root exist, each explicitly approved by the owner in-session; NOW.md points to the map session.

## Notes

- A frame is one session. Depth comes from map and shape, not from a longer frame.
- If the owner's idea is really a goal inside an existing direction, say so and route it there as a capture instead of creating a direction.
- frame never invents tree children, never creates tasks, never plans — that protects the owner's right to co-create the global plan in map.

# Play: frame

Purpose: create a new direction, or revise an existing direction's foundation. The only entry point for new directions and new top-level goals (gate G8).

Reads: owner's intent; for revision — CHARTER.md, TREE.md.
Writes: CHARTER.md, TREE.md, NOW.md, LOG.md.

## Steps

1. **Interview** — ask only what is missing, in one batch (≤7 questions): What outcome makes this direction worth years of your life? Success in numbers? Hard constraints (time/money/health)? What does failure look like? Which competencies does this direction need (these become lenses)? Which product repos exist or will exist?
2. **Charter** — draft CHARTER.md: mission (1–2 sentences), 2–4 measurable success criteria, constraints, lenses (default: product, audience, business — adjust to the direction), repos. Show to owner; iterate in-session.
3. **Pre-mortem** — "this direction failed three years from now": list ≥5 distinct failure reasons. Map each to a mitigation in the charter, a kill_by candidate, or an explicit accepted risk. Write into CHARTER.md.
4. **Outside view** — before drawing the tree, name 2–3 reference cases (how do attempts like this usually go, what base rates apply) and state what that implies for shape and sequencing. One paragraph in CHARTER.md.
5. **Root tree** — TREE.md: the root node (the mission as an outcome) and 2–5 child outcomes. Each child: goal stated as a verifiable change in the world, done_when, status `parked`. No deeper levels, no tasks (gate G2). Everything else the interview produced goes to `parked` nodes or captures.
6. **First bet candidate** — recommend which node to shape first: the one that tests the direction's riskiest assumption cheapest. Decision to owner (G7: options + recommendation).
7. **Close** — RESULT: state_changes creating the files, log line, next = CALL for a shape session on the chosen node (or awaiting_decision).

## Done when

CHARTER.md and TREE.md exist and the owner has approved them in-session; NOW.md contains the next CALL.

## Notes

- A direction is framed in ONE session. Depth comes later from shape cycles, not from a longer frame.
- If the owner's idea is really a goal inside an existing direction, say so and route it there as a parked node instead of creating a direction.

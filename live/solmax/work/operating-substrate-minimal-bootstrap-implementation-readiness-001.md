## Proof before live claims

Structural conformance:
- show the pack preserves Q1-Q7 and owns all legitimate local variation.

Useful live behavior:
- run a real owner-originated obligation end-to-end;
- return a useful result, bounded decision, clear block/failure or
  continuation without owner-operated internal bureaucracy.

Portable continuity:
- resume in a fresh run from the continuation without reconstructing the
  prior transcript;
- preserve governing definition, accepted basis, open obligations,
  current owners, authority, effect certainty and next action.

Failure/no-route:
- exercise at least one relevant adverse branch;
- avoid fabricated input and silent fallback;
- retain a named resolution owner and useful closure.

Effect safety, where effects are claimed:
- distinguish proposal, validation, authorization, apply and result;
- return authoritative effect evidence or explicit non-success;
- handle partial/outcome-unknown without false success or blind retry.

Governed evolution, before that claim:
- change a definition affecting active work and demonstrate an explicit
  compatibility, migration, pin/drain, handoff, block or unresolved
  disposition.

Demonstrated substrate reuse:
- at least two materially different packs preserve the same kernel
  semantics without kernel edits or consumer leakage.

These are proof categories, not a detailed test plan or selected harness.

# PLAY daily-review (local, находит — не чинит)

Триггер: расписанный запуск (Codex/Claude scheduled task) или фраза «дневное ревью» в любом чате на этом репо.
Назначение: раз в день сверить направление на консистентность ФАКТОВ и записать дельту находок. Сессия НИЧЕГО не чинит и не судит качество — только находит, классифицирует, приоритизирует, записывает.

Reads: NOW.md, LOG.md (свежий хвост), knowledge/g9c41-lanes-venues.md, work/review/findings.md, work/board/dashboard.html; git-факты OS-репо и продукта (fetch → tips веток, merged-статусы).
Writes (через RESULT, самописцем): work/review/findings.md (только дельта), панель (секции «Проблемы», «Журнал», «Борд»), LOG-строка, history-файл. Больше ничего.

## Steps

1. **Свежесть.** `git fetch` OS-репо и продукта; работать по committed состоянию. Чужие незакоммиченные правки в рабочей копии не трогать и не учитывать.
2. **Факт-сверки** (только проверяемое, БЕЗ суждений о качестве):
   - open_calls: deps «MERGED» — сверить по git, не по словам; базы CALL'ов против текущего origin/main (уехала — находка);
   - протухшие CALL'ы (issued давно, движения нет) и kill_by/даты ставок (близко/просрочены);
   - висячие указатели: SPEC/TREE/knowledge → NOW.decisions и файлы; LOG → history-файлы существуют;
   - END_OF_FILE-трейлеры state-файлов; бюджеты пьес (≤600 слов);
   - панель↔NOW: секция «Борд» не противоречит NOW (расхождение = находка «перегенерировать»);
   - канон-борд↔канон: C:\projects\gas_coop_game_canon\board\dashboard.html не противоречит файлам канона (CONSTITUTION/CORE/INDEX/QUEUE) и frame-слою OS (расхождение = находка класса night «перегенерировать канон-борд»);
   - чистота панелей: журнал держит ≤3 дней; «Проблемы» — только открытые; исторические секции-обсуждения помечены или вырезаны (история живёт в git, не на панели);
   - продукт: заявленные в NOW статусы веток (dev/dev2/core/lab) против фактических tips.
3. **Дедуп.** Каждая находка сверяется с work/review/findings.md по смыслу: known/rejected/fixed НЕ переоткрывать без НОВОЙ причины (причина цитируется). Показывается только дельта.
4. **Классификация** (решает, кто чинит):
   - `night` — ноль-поведение (доки/консистентность/tools-гигиена) → доступно ночной линии;
   - `repair` — дрейф состояния OS → в находке готовый текст repair-CALL;
   - `day` — поведенческое (код/гейты) → предложение дневного CALL через полный контур;
   - `maintenance` — дефект правил OS → черновик MAINTENANCE REQUEST (не сам фикс).
5. **Приоритет** P1 (блокирует работу/врёт состоянию) / P2 (мешает) / P3 (гигиена).
6. **Закрытие.** RESULT → применить самописцем: дельта в findings.md; панель — «Проблемы» (открытые, по приоритету), «Журнал» (LOG за последние 3 дня, сегодняшний день раскрыт), «Борд» (если протух); LOG-строка. Ничего не чинить. Budget ≤30 минут.

## Жёсткие границы

- Не редактирует NOW/TREE/CHARTER/пьесы/knowledge (кроме findings.md и панели).
- Не запускает фиксов, не открывает CALL'ы сам — только тексты-предложения внутри находок.
- Не судит дизайн/код-качество: находка без проверяемого факта не записывается.
- Переоткрытие rejected-находки без новой причины запрещено (урок reopen-tripwire).

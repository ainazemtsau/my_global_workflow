# PLAY refactor-night (local)

Триггер: владелец в любом чате на этом репо пишет «делай рефакторинг» (опции: «только ядро / доки / тесты / tools», «N штук»).
Назначение: автономная НОЧНАЯ линия рефакторинга с доказуемо нулевым изменением поведения. Ночью разрешены только headless-доказуемые зоны: `Core/**`, тесты, `tools/**`, `docs/**`. Адаптер/рендер/сцены — никогда ночью (это дневная линия B: нужен owner-eye).

Reads: NOW.md (open_calls → занятые зоны); knowledge/g9c41-lanes-venues.md (правила worktree/мержей); work/refactor/backlog.md.
Writes: продукт — только в ночных ветках `refactor/<slug>` в собственных worktree; OS — статусы бэклога + LOG + history через RESULT. **В main ночью не мержится ничего.**

## Steps

1. **Пред-полёт.** Свободные зоны = зоны кода без in-flight легов (по NOW.open_calls + git-фактам, не по словам). Занятые зоны исключить целиком. Правило 1 карты площадок (проверенный worktree) обязательно; ночные worktree создаются от свежего origin/main.
2. **Выбор.** Верхние элементы бэклога + открытые находки класса `night` из work/review/findings.md (минус явные исключения владельца в триггер-фразе), чьи зоны свободны; по умолчанию ≤3 за ночь (или N по слову владельца). Находки других классов (repair/day/maintenance) ночью НЕ брать никогда. Элементы вне бэклога/леджера брать нельзя — новые кандидаты только ЗАПИСЫВАЮТСЯ.
3. **Изоляция.** Каждый элемент — свой worktree/ветка `refactor/<slug>`; элементы независимы и идут параллельно.
4. **Доказательство (все пункты обязательны).** Код: голдены байт-в-байт; MeaningChecksum неизменен; полный `tools/check.ps1` зелёный; mutation не хуже базы; оба скана; diff строго в заявленных файлах элемента. Docs/tools-элементы: полный check зелёный + diff строго в заявленных файлах.
5. **Провал = откат.** Не доказалось, вылезло за заявленные файлы или потребовало поведенческого решения → откатить ветку, пометить элемент `failed:<причина>` в бэклоге. Никаких «почти получилось, оставил».
6. **G5-конвейер.** На каждый выживший элемент — G5-опровержение в свежей сессии («поведение изменилось? найди»), тоже ночью.
7. **Утренний отчёт.** Checkpoint RESULT: доказано / отклонено / ветки / статусы бэклога. Мерж в main — только владелец утренним батчем, через очередь мержей карты площадок (полный check на merged main на каждом).

## Жёсткие запреты

- Попутные улучшения и любое изменение поведения — ноль, точка.
- Переименования/правки в горячих файлах (`VoxelField.cs`, `VoxelFaceFlow.cs`) вне явно заявленного элемента.
- Checksum-поверхность (включая константные стабы вроде BiasChecksum) = DO-NOT-TOUCH.
- Мерж, push в main, правка чужих in-flight веток.

## Done when

Утренний отчёт отдан: каждый взятый элемент либо доказан (ветка + G5 могли-не-опровергнуть, ждёт владельческого мержа), либо честно откатан с причиной; бэклог и LOG обновлены через RESULT.

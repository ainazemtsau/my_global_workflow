# PLAY lanes-board (local, render-only)

Триггер: владелец в любом чате на этом репо пишет «что можем делать» / «борд» / «board».
Назначение: показать, что в работе, что можно запустить параллельно ПРЯМО СЕЙЧАС (и как), что заблокировано и чем, что ждёт владельца, и очередь мержей. ТОЛЬКО ЧТЕНИЕ: пьеса не меняет файлов, не эмитит RESULT (read-only exception, KERNEL §2), не открывает и не закрывает CALL'ы.

Reads: NOW.md; knowledge/g9c41-lanes-venues.md; git-факты OS-репо и продукта (`git -C C:\projects\Unity\GasCoopGame worktree list`, tips main/dev/dev2/lab/levels, origin/main SHA, наличие `Temp/UnityLockfile` в worktree'ях).

Steps:
1. Прочитать NOW.md (open_calls, next, tasks, decisions) + карту линий.
2. Собрать git-факты (список выше). Статусы deps проверять ПО GIT (merged/не merged), не по словам заметок.
3. Каждому open_call определить: линия, площадка, стадия (BUILD / review / ждёт мержа / ждёт владельца), занят ли его worktree.
4. Каждому CALL из next проверить запускаемость: линия свободна? deps MERGED? worktree назначен и свободен (правило 1 карты — жёсткое правило владельца)? база актуальна?
5. Отрендерить борд владельцу по-русски, без жаргона:
   — В РАБОТЕ: линия → CALL → площадка → стадия.
   — МОЖНО ЗАПУСТИТЬ СЕЙЧАС: CALL → готовая пошаговая инструкция запуска (какой чат/репо, какой worktree/редактор или headless, что написать первой строкой).
   — ЗАБЛОКИРОВАНО: CALL → чем именно (dep / занятый worktree / решение владельца).
   — ЖДЁТ ТЕБЯ: батч гейтов и вердиктов с оценкой минут.
   — ОЧЕРЕДЬ МЕРЖЕЙ: слоты.
6. Ничего не записывать. Найденное расхождение (dep числится merged, а в git его нет; worktree занят, а CALL помечен READY; база уехала) — показать в борде с пометкой ⚠ и порекомендовать repair-CALL, но не чинить самому.

Budget: ≤15 минут, читающая сессия. Язык владельца: русский.

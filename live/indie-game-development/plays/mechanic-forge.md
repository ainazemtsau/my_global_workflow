# Play: local/mechanic-forge

Purpose: forge ONE game-mechanic question (a "how does it PLAY" node, `area: mechanic`) from open to a **worked,
playable loop** — owner-chosen, developed VERB-FIRST, gated by the mechanic lenses. UNLIKE `canon-forge` (lore mode,
which freezes a compact invariant card), a mechanic's value is the EXPANDED moment-to-moment loop; the one-line label is
only an index. Mechanics are CO-CREATED: the AI proposes/develops, the owner decides; the AI never freezes invented
mechanics. Mechanics GROW FROM the gas core + co-op extraction; lore is a thin late label, never a justification.

Reads: `live/indie-game-development/knowledge/mechanic-lenses.md` (the BINDING 6-lens gate — read it in full); the canon
repo `C:\projects\gas_coop_game_canon` frozen parents the mechanic must grow from (`q-north-star` + the relevant
mechanic/world cards: `q-gas-role`, `q-the-city`, `q-base-logistics-labs-factories`, sibling `q-coop-shape`); prior
research in `work/` (e.g. `research-expedition-loop-foundation-2026-06-19.md` — RAW MATERIAL, not answers); `TREE.md`
(node `g-d3a8`).
Writes: the CANON REPO — the frozen mechanic card as a WORKED LOOP (verb table + numbered second-by-second cycle in
units/seconds + a separated-player scenario; frontmatter `status / depends_on / lenses{}`; the one-line label = index,
NOT the deliverable), the `rejected_alternatives` tail, the child questions, the area MOC. In OS state: `LOG.md` (one
line). Does NOT touch `NOW.md` or the active bet (canon runs PARALLEL, never a second active bet — G1).

Precondition: a mechanic question chosen (its `depends_on` parents frozen). Reached via CALL `to: session, play:
local/mechanic-forge`. One question per session (one chat = one job).

## Steps
1. **Frame the REAL question (plain words) + decompose** `(owner)` — state, in plain words with no jargon, what we are
   actually deciding; if the node is secretly several questions, SPLIT it before any options. Get the owner's agreement
   it is the right question and where the node sits in the graph. No A/B/C yet.
2. **Verbs before names** — list what the player physically DOES: `verb / object / input / perceivable consequence (in
   units & seconds) / players-required-simultaneously`. NO name is proposed until a verb row with a units/seconds
   consequence and a 2+-player requirement holds. The name is an index, attached last.
3. **Develop a worked loop + spar** — build the moment-to-moment loop as a named, second-by-second scenario with
   separated players, NOT a slogan. Stake and defend a concrete position; develop the owner's fragments by adding NEW
   verb-consequence (reorganizing his words is a non-answer). Critique your own and the owner's ideas equally; keep only
   what survives mutual critique.
4. **Gate — run `knowledge/mechanic-lenses.md` in order** — 1 grows-from-core (anti-contrived) → 2 interesting-decision
   → 3 anti-solo → 4 pillar-pull → 5 field-read → 6 grey-box. Lenses 1–5 are paper = status **UNVERIFIED** ("not
   contrived, can-prototype"); a fail returns to step 2/3 (a FIX re-enters at lens 1). Lens 6 (the un-fakeable 2-player
   grey-box) is the ONLY "fun" verdict (**CERTIFIED**) and is usually a later, owner-run step.
5. **Freeze & grow** `(owner)` — owner signs → freeze the WORKED LOOP as the card (label = index); log dropped
   directions as `rejected_alternatives`; spawn child questions; update the MOC. Record which lenses passed on paper vs
   which still await the grey-box (the honest UNVERIFIED→CERTIFIED line). Numbers in the loop are TUNING-PENDING
   placeholders (shape is canon, magnitudes are not).

## Rules of engagement (the failures this play exists to prevent)
- No premature A/B/C option-machinery — find the real higher question first.
- No compressing a mechanic to a slogan; no invented umbrella jargon ("gas contract"-style names).
- Don't rephrase the owner's words — add concrete NEW verb-consequence, or it is a non-answer (rejected unread).
- This is a ONE-ON-ONE spar, NOT a fan-out; spawn subagents only if the owner asks for a bounded research sub-question.
- "The sim runs" is not a milestone; "two players FEEL it" is. Paper never certifies fun — only the grey-box does.
- Co-creation is binding: never freeze invented mechanics; the owner decides.

## Done when
One worked playable-loop mechanic card is frozen in the canon repo (owner-signed), gated by the lenses (paper pass
recorded, grey-box status honest); child questions spawned; a `LOG.md` line written.

## Notes
- Mechanics-first is the DEFAULT; lore LEADS only for the rare flagship identity-seed (the delete-test in lens 1). One
  web with the lore graph, not two trees.
- The deliverable is the LOOP; the invariant/label is a derived index, never the whole card.

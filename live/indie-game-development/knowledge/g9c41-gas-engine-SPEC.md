## 2. Залоченная архитектура (7 фактов)

Ратифицировано владельцем «да» 2026-06-22 (`d-arch-lock-slices-001`).

> **🧭 ВЕДУЩИЙ КАДР (читать ПЕРВЫМ — он определяет, как понимать все 7 фактов).**
> Архитектура = **КООРДИНИРУЮЩИЙ GRID + СЛОИ**, НЕ одна газовая сетка, из которой выводится всё.
> - **GRID** = общее координатное/адресное пространство + карта владения `cell→region/sector` + шина событий (publish-events) + такт-ревизия (commit clock). Он КООРДИНИРУЕТ, но НЕ хранит данные слоёв и НЕ роутит потоки (не god-object).
> - **СЛОИ** (газ 50см / структура 25см / размещение 25см-зарезервировано) — каждый держит СВОИ данные на СВОЁМ разрешении и разметке. Кросс-слойное чтение — ТОЛЬКО против ЗАФИКСИРОВАННОЙ (committed) ревизии чужого слоя через read-model (лаг 1 тик; НИКОГДА live mid-tick — это и есть инвариант §9.7).
> - **СВЯЗЬ слой→слой = детерминированная целочисленная проекция** (структура 25см → проводимость газовой грани 50см, §9.9), односторонняя и статическая. Газ остаётся одной 50см-сеткой (нет газ↔газ re-flux шва).
> - **Деталь-пузырь у игрока = КАПНУТ фиксированным радиусом-сферой** (off-checksum, см. Факт 3; стартовый ADR 4м@50см) — закрывает «зал» по визуалу; радиус+размер-клетки лочатся ADR.
> - **Большие залы (отложено, расширяемо):** дефолт — ~однородные малые/средние комнаты; гигантский зал НЕ спец-кейсится сейчас. Зарезервировано (НЕ активно): авторить зал как сегменты-комнаты с дверьми-во-всю-стену (дальнее грубее даётся даром) ИЛИ дальний вертикальный LOD `mid-2.5D` (§7, far-only) + sub-partition. Включаем ТОЛЬКО если замер покажет проблему.
> Это НЕ новое изобретение, а ПОДНЯТИЕ в ведущий кадр того, что уже живёт в §9-швах: §9.4 (collapse/expand по-слойно), §9.5 (чексумм по типу/СЛОЮ), §9.7 (committed-ревизия read-model + grid = общее адресное пространство и шина событий, НЕ роутер), §9.10 (запас адресов под доп. реплицируемые слои).

> 📌 **Формальная ратификация лока (важно).** Формулировка Факта 2 РЕ-ШЕЙПНУТА 2026-06-26 (owner «да», см. `d-cellsize-split-001` / R1-layered-reshape): GRID = координирующий каркас; ГАЗ 50см ИЗОТРОПНО авторитетно (высота ЭМЕРДЖЕНТНА, БЕЗ полос — сохранено); СТРУКТУРА/РАЗМЕЩЕНИЕ = слои на своём разрешении (дефолт 25см), связь = целочисленная проекция (§9.9). Это owner-signed lock-change (G9) поверх `d-cellsize-ratify-001` (исходный 50см-изотропный лок СОХРАНЁН для газа — ре-шейп его не отменяет, а поднимает слойный кадр). **Подпись владельца на этом спеке = G9-ратификация** этой правки лока (операционно она уже в силе через c-exec-013 + баннеры; формальная правка текста лока ждала твоего «да» — и ЭТОТ спек его собирает). **Факт 1. Нетcódigo = input-lockstep.** По сети идут ТОЛЬКО нажатия игроков; каждый пир детерминированно пересчитывает весь сим сам. Условия детерминизма: целочисленная/фикс-арифметика + сидированный RNG + закреплённый порядок обхода + по-тиковый чексумм по СМЫСЛУ; рассинхрон = громкая остановка. Предел масштаба = CPU самого слабого пира, НЕ канал. Миграция хоста почти бесплатна. (Лок = **ADR-0002**; вытесняет host-broadcast путь ADR-0003/0004 (+ADR-0005 в части репликации), окончательно retired ADR-0016/S2a. NB: ADR-0010 = тест-песочница, НЕ лок — см. поправку §1.)

**Факт 2. КООРДИНИРУЮЩИЙ GRID + СЛОИ НА СВОЁМ РАЗРЕШЕНИИ; ГАЗ = 50см ИЗОТРОПНО, авторитетно. Высота ЭМЕРДЖЕНТНА, БЕЗ полос.** *(re-shape 2026-06-26, owner «да», R1-layered-reshape; gas-50 лок сохранён)*
- **GRID — это КООРДИНИРУЮЩИЙ каркас, НЕ единая 50см-модель данных и НЕ god-object/роутер:** общее координатное/адресное пространство + карта владения `cell→region/sector` + шина событий + такт-ревизия (commit clock). Данных газа он не хранит и потоки не считает. Газ / структура / размещение — это первоклассные СЛОИ; каждый держит свои данные на СВОЁМ разрешении и общается с другими ТОЛЬКО через committed-ревизию (см. §9.7). «Одна 50см-модель на всё» — это конфляция: 50см — разрешение ГАЗА, не универсального грида.
- **Слой ГАЗ = 50см глобально, авторитетно везде**, по ВСЕМ осям, включая вертикаль (клетка = куб 0.5×0.5×0.5 м, 8 клеток/м³). Это ЕДИНОЕ разрешение симуляции газа (нет помеси размеров газа по комнатам/тирам, нет газ↔газ re-flux шва) — заратифицированный выигрыш цены, НЕ ослабляется.
- **Слой СТРУКТУРА = 25см (дефолт), грани** — единственный источник структурной правды (стены/полы/проёмы/пробиваемость). Тоньше газа НАМЕРЕННО. Связь со слоем газа — детерминированная целочисленная проекция ВНИЗ (§9.9): 50см газовая грань делится на 2×2 = 4 под-грани 25см; проекция отдаёт ПО-под-грани occupancy (битмап), а газ читает счёт открытых под-граней В ПРЕДЕЛАХ грани СВОЕЙ Z-клетки (не один скаляр на всю стену — см. §9.9). Односторонняя (структура→газ), статическая, целочисленная. Авторится метками на переиспользуемых модулях (см. §9.9/§5): ГЕОМЕТРИЧЕСКИЕ метки — владелец ИЛИ билдер-через-MCP, валидируются против геометрии; СЕМАНТИЧЕСКИЕ метки (материал/порог/маска) — ТОЛЬКО владелец/фикстура (нет геом-оракула → билдер не авторит).
- **Слой РАЗМЕЩЕНИЕ = 25см, точки** — РАЗРЕШЕНИЕ ЗАРЕЗЕРВИРОВАНО, механика ОТЛОЖЕНА (объектов пока нет; адресный запас — §9.10).
- **Высота эмерджентна из геометрии**, а не хранится как полосы. Низкая дверь сливает только нижние клетки газа (газ выше порога не переходит). «Присесть под слой» работает БЕСПЛАТНО: голова и ноги стоят в разных Z-клетках газа 50см, урон читает Z-клетку головы. Стопка комнат объём-нейтральна (3 этажа = 3 в ряд = тот же объём, тот же счёт газовых клеток).
- 25см в СЛОЕ ГАЗА не вводится: тоньше-25см там — ТОЛЬКО локальный off-checksum визуальный пузырь у игрока/инструмента, НИКОГДА размер комнаты и НИКОГДА авторитетный путь газа. (25см слоёв структуры/размещения — это ДРУГОЕ: их собственное авторитетное разрешение, не газовый пузырь.)
- Две оси LOD газового слоя: (а) размер клетки геометрии-проекции и (б) пространственный room-rollup далёких комнат (S4). Комната = **МЕТКА/раздел (region_id) на КАЖДОМ тире и в КАЖДОМ слое** (труба-портал вблизи убрана, метка-раздел остаётся везде, она уже в чексумме). collapse/expand — единственный путь конверсии (discard-on-leave).

**Факт 3. Деталь = ЛОКАЛЬНОЕ неавторитетное уточнение грубой авторитетной правды.**
- Грубый (покомнатный) сим = ЕДИНСТВЕННАЯ общая правда, считается одинаково на каждом пире, входит в чексумм.
- Детальный «пузырь» считается ТОЛЬКО там, где СВОЙ игрок пира (≈1 пузырь/пир, не N → ≈×8 выигрыш CPU), он ВНЕ чексуммы, НИКОГДА не пишет обратно в грубое, при выходе из комнаты ОТБРАСЫВАЕТСЯ (не сворачивается в авторитет). **Пузырь КАПНУТ фиксированным радиусом-сферой** (не на размер комнаты) — закрывает «зал» по визуалу БЕЗ касания авторитета/массы/детерминизма (это единственная адаптируемая часть «облачной» идеи владельца; clouds-as-authority отвергнуты — cloud-vs-grid анализ 2026-06-26); радиус+размер-клетки ЛОЧАТСЯ ADR (старт 4м@50см; 25см-внутри или большой радиус тихо вернут взрыв → лок обязателен).
- Жёсткие общие последствия (реакции/урон/межкомнатный поток/взрывы) ПРОМОУТЯТСЯ в **грубые СОБЫТИЯ**, считаемые из грубого состояния на каждом пире (входят в чексумм). Ноль нового сетевого трафика — по проводу только инпуты.
- Локальный страж: пир каждый тик проверяет `collapse(деталь)==грубое-напрямую` и в чексумм подаёт только согласованное значение.
- Глубина каждой механики классифицируется на КАЖДОМ PLAN слайса в ведро-1/2/3 (см. §3). Дефолт = покомнатная глубина (Вариант A). **Реклассификация монотон-B → A (variant-B анализ 2026-06-26):** B-механика, ДОКАЗАННО монотонная/path-independent (медленная заливка, fill-line), решается порогом по in-checksum грубому тоталу (`сломался = накопленная_масса ≥ порог_угла`; ноль детальных клеток, точнее столбиков) — под RED-оракулом с не-монотонным контролем (ветер в угол и обратно), который ОБЯЗАН провалиться. Это сжимает число НАСТОЯЩИХ-B комнат — главный рычаг доступности B. Настоящая-B (не-монотонная) комната = full-3D. `settled_flag` (целое в чексумме, ticks-since-net-flux) гейтит ТОЛЬКО far-демоут/сон, НИКОГДА near-переход.

**Факт 4. Разреженная доминантная модель газа + целочисленная химия.** В клетке: доминирующий тип + количество + малый ФИКСИРОВАННЫЙ ограниченный mix-overlay на АКТИВНУЮ клетку, инлайн/кэш-дружелюбно (ПОВЫШЕНО 2026-06-30 из «временный mix-overlay при реакции» → постоянный ограниченный per-active-cell слот; НЕ плотные страницы на каждый газ, НЕ per-cell hashmap). Главная экономия — sparse-awake через активный фронт. Химия = целочисленная таблица столкновений (типы/реакции/телеграф/спец-поведение). **Принцип «как ДАННЫЕ, не как код-ветвление»** (УТОЧНЕНО 2026-06-29 — НЕ «всё уложено в S0 day-one»): каждый кусок кладётся как ДАННЫЕ-СХЕМА на СВОЁМ слайсе — per-тип параметры + per-cell TypeId на **Sc-types**, таблица реакций на **Sc-reactions** — и per-cell TypeId входит в meaning-чексумм с момента ввода типов (schema-решение ТАМ, не миграция чексумм-схемы на живых узлах). **Детерминизм — как гарантируется (`d-reflux-gate-001`):** целочисленный авторитетный путь + build-time **zero-float scan** + **loopback-хеш на ОДНОЙ машине** (2 процесса). Целочисленный кросс-CPU = ДАНО (как в банках), его не доказывают. Реальные 2 ФИЗИЧЕСКИЕ машины — НИКОГДА не гейт (у владельца одна машина — достаточно); опциональный одноразовый прогон — максимум. Float в авторитетном пути ловит zero-float scan на одной машине — ЭТО и есть кросс-CPU гарантия по построению. Single-owner-per-face + gather-then-apply + канонический порядок + сидированный RNG. **Целочисленный кламп массы (несущее — скан float его НЕ ловит):** каждый gather-then-apply ОБЯЗАН жёстко кламповать массу в [0,cap]; отрицание/переполнение = громкий assert, НИКОГДА тихий wrap — wrap бит-идентичен на всех пирах → чексумм ЗЕЛЁНЫЙ, а физика мусор.

## 3. Модель газа

**Клеточная модель.** 50см изотропно, целочисленно, разреженно (доминантный тип + количество + малый ФИКСИРОВАННЫЙ ограниченный mix-overlay на активную клетку — ПОВЫШЕНО 2026-06-30 из «mix-overlay при реакции» до постоянного ограниченного слота, см. Факт 4). Спящий насыщенный газ хранится байт-сном (lossless), считается только активный ФРОНТ.

**Закон face-flow (вблизи).** Газ течёт клетка→клетка через ОТКРЫТЫЕ ГРАНИ; стена = закрытая грань; дверь = набор открытых граней. Из геометрии БЕСПЛАТНО, в день один, выводятся три вещи:
- **R6: площадь проёма ВСЕГДА влияет на поток** — площадь = количество открытых граней (узкая щель сочится >0; не «всё или ничего»).
- **высота эмерджентна** — низкий проём сливает только нижние клетки.
- **сквозь стены нет** — структурно, из геометрии.
Перенос = целочисленный gather-then-apply + carry (остаток). Стена = 0 потока, щель = поток >0. (Это строго лучше старого скалярного портального закона `t=mEq/kP`, который площадь не умеет.)

**Ёмкость + back-pressure (КЕПТ-закон, `d-roomfull-001`):** поток/инъекция клампится в [0,cap]; источник idle при полной комнате; насыщенный закрытый регион saturate-and-hold, масс-точно. §6 п.1 `cap` = ОТДЕЛЬНЫЙ опциональный доп-потолок, НЕ этот инвариант; сток в этой волне НЕ вводим.

**Форсированный поток (S1 СДАНО, c-exec-015).** Поверх градиентного face-flow работает слой ИМПУЛЬС-СОБЫТИЙ: детерминированное целочисленное событие `{грань-footprint, направление, сила, ttl}` от инпута игрока пишет ЗАТУХАЮЩИЙ направленный bias-регистр на открытую грань (in-checksum, по-грани, по-грани-УНИФОРМНО — §9.5 trip-wire; НЕ хранимая скорость). Реализованные случаи: ВЫБРОС (источник с направленным толчком, потом оседает), ВЕТЕР (устойчивый импульс несёт газ по коридору + делит на развилке), односторонний КЛАПАН (designed permanent split, реверс-поток клампится в 0), jet-emitter (толчок вперёд + боковое сжатие). Устойчивость СОХРАНЕНА: контракция (Kp≥2*degree) приостановлена в форс-окне, RED-оракул гарантирует возврат к покою после затухания; симметричный целочисленный decay; ±1 floor (слабый bias не молчит). НЕ истинная инерция/coasting (это резерв TIER-3 — `d-gas-richness-tiers-001`). ⚠ **Разделение локов** (аудит 2026-07-02 multiplayer-06): TIER-3 (fixed-point velocity) вскрывает НЕ netcode-лок ADR-0002, а границу ПРЕДСТАВЛЕНИЯ «нет хранимой скорости» (S1 / ADR-0012-s1). Целочисленная fixed-point скорость СОВМЕСТИМА с lockstep (прецедент Photon Quantum/StarCraft) — будущий вердикт по богатству R1 НЕ должен спутать это с ре-открытием R6-фундамента. Оба boundary вскрываются только подписью; это РАЗНЫЕ boundary. ВЫДАВЛИВАНИЕ (S1b) ВЫРЕЗАНО в бэклог (по умолчанию косметично; `d-displacement-s1b-cut-001`).

**Вертикаль вблизи = buoyancy-BIAS, НЕ сортировка.** Подъём/оседание = целочисленный bias на поток через Z-грани (ползущий фронт). Пер-колонка мгновенная сортировка по высоте ЗАПРЕЩЕНА вблизи (телепорт тяжёлого на пол = вертикальный поп, ровно то, что no-pop запрещает). Layer-sort — только дальний/грубый тир (текущий `CoarseBandStep` сортирует — НЕ копировать его в near).

**Near / Far (одна модель, два гранулярности).**
- **Near** = 3D-грид 50см, поток через открытые грани; портал-как-труба вблизи НЕ нужен (связность неявна в состоянии граней).
- **Far** = room-graph **ROLLUP** над ТЕМ ЖЕ разделом; портал = агрегат потока по cut-set открытых граней. Грид и граф — один закон сохранения на двух гранулярностях (грид = базовая таблица, граф = материализованный rollup над ней; rollup требует РАЗДЕЛА = граф-как-метка).
- **Комната = метка (region_id) на КАЖДОМ тире.** Слово «портал» делало две работы: труба-переноса (вблизи УДАЛЕНА, её заменяют грани) и метка-раздел (СОХРАНЕНА ВЕЗДЕ — нужна для границ collapse/expand, рёбер пролома, читателя-вокселайзера, под-раздела ангара, сортировки чексуммы по region_id, суммирования при детонации).
- **collapse(region):** `coarse_mass[layer]` = точная целочисленная СУММА клеток; температура = `{energy_sum, capacity_sum}` (НЕ предделённое среднее); остатки граничных граней → coarse-edge flux-register; открытые граничные грани → cut-set портала. **expand(region):** largest-remainder, сумма бит-точна. Два «no-pop»: (1) поток через дверь идентичен, fine сосед или coarse (через re-flux); (2) форма/состояние комнаты, куда вернулся (D5/удержанные комнаты), хранит разреженную fine-ТЕНЬ → expand = тождество; далёкие/мелкие комнаты регенерируют форму.

**Деталь = локальное + грубые СОБЫТИЯ.** См. факт 3. Поток данных однонаправлен: деталь читает грубое, никогда не пишет обратно; при выходе — discard. Жёсткие последствия = грубые события из грубого состояния на каждом пире.

**Фронт = ПРОИЗВОДНЫЙ вид, не отдельное поле (R21).** Авторитетна МАССА: она наступает/отступает/сохраняется целочисленным переносом. «Фронт» = отображаемая передовая непустая клетка/комната, ПЕРЕСЧИТЫВАЕТСЯ из массы, ВНЕ чексуммы — НИКОГДА не второе реплицируемое «сохраняющееся» поле (это отвергнутый Wave-3 вариант). Сохраняющаяся величина = масса, не фронт. Граф/Dijkstra-фронт осмыслен ТОЛЬКО на far-тире (вблизи граф-транспорта нет — его заменили грани).

**Что БЬЁТ и что ТРИГГЕРИТ — читает ГРУБОЕ, не пузырь (несущие инварианты).** Урон/контакт/доза читаются ТОЛЬКО из грубого авторитетного поля на каждом пире, НИКОГДА из приватного пузыря (доза = время-интегрированное грубое событие из резидентного газа; 0 overlap → 0 доза). Триггер реакции/детонации читает refinement-инвариантные ГРУБЫЕ тоталы, НЕ пост-распределённые fine-клетки (иначе подход игрока сам детонирует облако = gameplay-pop + рассинхрон); цепь — следующим тиком.

**Ведро-классификация (на каждом PLAN, дефолт Вариант A — покомнатно):**
- **Ведро-1 — сглаживается:** свободное локальное ощущение (вихрь; раздвоение, которое сливается обратно в комнате) — ВНЕ чексуммы, в отбрасываемом пузыре.
- **Ведро-2 — грубо-промоутируемо:** дешёвое общее СОБЫТИЕ из грубого состояния, для всех, В чексумме (триггер реакции + масса; межкомнатная вентиляция; урон-где-газ; маршрутизация по высоте через несколько вертикальных слоёв грубого тира).
- **Ведро-3 — под-комнатное И общее:** дорогой случай (стойкое раздвоение, меняющее выходную дверь; точный прорыв канала, решающий исход) → «эта комната считает деталь для ВСЕХ пиров» или рассинхрон. Берётся осознанно, только в нужных комнатах. Первый проход по примерам владельца: в основном A + 2-3 ведра-3.

**Вариант A vs B (фундаментальнее сети, решает владелец на каждой механике):** A = глубина НА УРОВНЕ КОМНАТЫ (3D-деталь = ощущение сверху; деталь может быть даже float/недетерминированной — отбрасывается; страшная re-flux-проба ПАДАЕТ до owner-eye). B = глубина в ПОД-комнатной ФОРМЕ (точная форма облака решает исход) → для таких механик дешёвая модель НЕБЕЗОПАСНА, форсит ведро-3. SS13 в основном покомнатна и при этом глубока.

**Глубокое уровне-охватное взаимодействие — ПОДДЕРЖАНО** (доказано каноном + SS13), это ЦЕНТР архитектуры; фаза = BUILD, не research.

## 4. §9 швы (данные day-one, заглушки, ноль поведения)

Все швы кладутся как ФОРМА ДАННЫХ с первого дня (S0), реализация — позже. Load-bearing, воспроизведены полностью.

1. **Площадь проёма в законе потока.** Сегодня `OpeningSize` существует, но ИГНОРИРУЕТСЯ (закон `t=mEq/kP` без площади). Шов = вписать площадь (= count открытых граней) в закон переноса.
2. **`representation_tag` + `resolution_tag` на регион.** `representation_tag` = переключаемое множество `{fine-3D | coarse | bucket}` (mid-2.5D — superseded как near-авторитет, см. §7; far-only вертикальный LOD — отложено-резерв под большие залы, см. Факт 2 / ведущий кадр); для S0 = `fine-3D`. `resolution_tag` = ПАРА `(geometry_res, gas_res)`; **дефолт `(25, 50)`** — геометрия/структура ТОНЬШЕ газа, переключаемо через ТОТ ЖЕ тег (без пересборки; доп. config-машинерию НЕ вводить). **Инвариант (ИНВЕРТИРОВАН 2026-06-26):** `gas_res` обязан быть целым кратным `geometry_res` (газ НИКОГДА не тоньше структуры; 50/25 ⇒ кратность 2). geometry_res конфигурируем (тоньше-25 позже — приятная возможность, НЕ обязательна); дефолт 25, не зашит. Связь слоёв = детерминированная целочисленная проекция структура→газ (см. §9 шов 9). Старая формулировка «geometry = целое кратное gas, геометрия НИКОГДА не мельче газа» ОТМЕНЕНА этим ре-шейпом (она строилась под обратный, далёкий тир «грубее: 50→100»; грубее-100 остаётся допустимым ВЕРХНИМ концом диапазона как config, не дефолт).
3. **Целочисленный flux-register (остаток/carry) на КАЖДОМ ребре** с первого дня (даже fine-to-fine) → все будущие coarse/fine швы и sleep/wake без потери массы.
4. **`collapse()/expand()` = ЕДИНСТВЕННЫЙ путь конверсии** — чисто целочисленный, по-слойный (НИКОГДА не усреднять колонку в один скаляр), связан с инвариантом остатка. В S0 — только тело-ТОЖДЕСТВО.
5. **Чексумм покрывает СМЫСЛ, не значение** — включает: грубую массу по типу/слою, region_id, состояние граней, promo-события, `TypeId` слота, живой размер-клетки/тир, page-id, канонический порядок awake-очереди, дозу лута (D6). **+ под sparse-dominant (Sc-rep, 2026-06-30):** per-cell ШТАМП доминирующего типа (МУТАБЕЛЬНЫЙ) + ограниченные mix-overlay-слоты, фолдятся **cell-keyed** (канонический X,Y,Z — НЕ bag/сумма/сорт-по-значению) — schema-решение СЕЙЧАС, не миграция на живых узлах; loopback-only до сети ~S4. (exposure-аккумулятор динамической типизации — ОТДЕЛЬНЫЙ будущий член на слайсе реакций/типизации, не здесь.) Вся деталь — ВНЕ. **Слой структуры:** в S0 проекция структура→газ count-only и БЕЗ потери от идентичности (ни один эффект не читает, КАКИЕ под-грани открыты) → в чексумм входит спроецированный счёт conductivity, а сам occupancy-битмап под-граней — ВНЕ. **Trip-wire (инвариант):** ПЕРВЫЙ слайс, где какой-либо эффект зависит от того, КАКИЕ под-грани открыты (направленный пролом, асимметричное расширение, клапан на нижней половине, direction-mask) — ОБЯЗАН СНАЧАЛА сложить occupancy-битмап под-граней в meaning-чексумм (канонический порядок: sub-face index внутри газовой грани), иначе два пира с одинаковым счётом, но разным битмапом, разойдутся при зелёном чексумме (повтор «чексумм ЗЕЛЁНЫЙ, физика мусор», Факт 1) на уровень выше. Это schema-решение принять СЕЙЧАС, не как миграцию чексумм-схемы на живых узлах.
6. **Температура = `{energy_sum, capacity_sum}`** (целочисленный аккумулятор из двух полей, НЕ предделённое среднее) — для идемпотентного collapse; эффективная температура = energy_sum/capacity_sum, никогда не хранится предделённой.
7. **`FluxAcross(opening, rev)`** (refinement-инвариантный целочисленный поток) + ОТДЕЛЬНЫЙ хеш player-kinematics (хеш газа не ловит движение игрока) — для будущих силовых механик; решить ДО первой силовой механики. **Межслойное чтение — ТОЛЬКО против ЗАФИКСИРОВАННОЙ (committed) ревизии в grid-координатах** (publish-events + pull read-model); слой НИКОГДА не читает чужое live mid-tick состояние — same-tick кросс-слой = рассинхрон (1-тик лаг и есть причина, почему отложенная temp→gas обратная связь вскрывает лок).
8. **Страж `(byte)c` в `ChunkEncoder`** — починить тихое переполнение (>255 чанков, хеш зелёный). *Прим.: на dev уже защищён (`ChunkEncoder.cs:46-56`) — СОХРАНЁН-уже-защищён, не передоказывать.*
9. **Атрибуты ребра** — ЗАРЕЗЕРВИРОВАТЬ: direction-mask (односторонние клапаны), целочисленную conductivity, порог взрыва. **Целочисленная conductivity = ПРОЕКЦИЯ структура→газ (определена, не просто запас):** слой структуры на 25см; одна 50см газовая грань делится на 2×2 = 4 под-грани 25см. **Структура хранит per-под-грань occupancy-БИТМАП (какие 25см под-грани открыты), не только счёт** — это авторитетный датум структуры. Газ читает ПРОЕКЦИЮ: `conductivity = число открытых 25см под-граней В ПРЕДЕЛАХ грани его Z-клетки`. Проекция считается ПОСЛЕ декомпозиции на Z-клетки газа: низкая щель проецируется на грань НИЖНЕЙ Z-клетки (conductivity>0) и грань ВЕРХНЕЙ Z-клетки (conductivity=0) РАЗДЕЛЬНО — НИКОГДА один скаляр 0..4 на всю вертикальную стену (иначе низкая щель и высокая фрамуга с одинаковым счётом алиасятся → нарушение «низкий проём сливает только нижние клетки», Факт 2). Для грани, НЕ делимой границей Z-клетки, диапазон = {0..4}; для делимой — счёт берётся внутри каждой Z-полосы. **`conductivity` — это целочисленный СЧЁТ (0..N открытых под-граней), НЕ делёная доля: НИКОГДА не хранить `openSubFaces/4` (ни float, ни целочисленное деление — последнее схлопывает {1,2,3}→0 и душит щель, нарушая R6, шов 1).** «¼-проводимость» — описательно. Проекция ОДНОСТОРОННЯЯ (структура→газ), СТАТИЧЕСКАЯ ВНУТРИ тика, ЦЕЛОЧИСЛЕННАЯ — детерминизм сохранён, газ остаётся одной 50см-сеткой (нет газ↔газ re-flux шва). В S0 — present-but-stubbed (открыто ⇒ полный счёт, закрыто ⇒ 0); реальная sub-face проекция включается на слайсе структуры/пролома. **Шов разрушаемости:** per-room back-pressure = ХРАНИМОЕ число давления, которое читает будущая механика «давление ломает дверь по прочности» (`d-roomfull-001`) — это шов, а не только порог; базовый пролом (S5) = открыть ЛАТЕНТНЫЕ 25см под-грани (флипнуть occupancy-битмап структуры) → ре-проекция → смена спроецированной conductivity → поток газа. **Изменение спроецированной conductivity (пролом) на границе СВЁРНУТОГО/удержанного региона (D5 / ведро-3) обязано проходить через существующий re-flux-гейт collapse/expand (`d-reflux-gate-001`, шов 3/4), а не трактоваться как «статическая проекция»: пролом ТРИГГЕРИТ re-flux, но сам остаётся газ↔газ событием на 50см-границе, НЕ вводит газ↔газ переток через 25/50-границу.**
10. **Запас адресов plan/page** — если понадобятся доп. реплицируемые слои (фронт, по-видовая температура).

**4 GAP-сокета (день один в S0, в чексумме):** GAP-1 cap (`max_mass/rate_limit/dissipation`, заглушка ∞/0); GAP-2 void/exterior region_id (сток); GAP-3 `resolution_tag` = ПАРА `(geometry_res, gas_res)` дефолт `(25, 50)` (геометрия тоньше газа; инвариант = gas_res целое кратное geometry_res; проекция = шов 9); GAP-4 per-region `tick_divisor` (=1).

## 5. Спайн слайсов S0–S6

Слайсы = задачи в катящейся волне, НЕ узлы дерева. Порядок = эффект × зависимость-фундамента. Глубина каждой механики (ведро-1/2/3) классифицируется на PLAN слайса, дефолт A. Ритуал слайса: PLAN (впитать+применить все решения, классифицировать механики) → RED-first независимый тест-автор → целочисленно-детерминированный zero-legacy build → гейт + G5 (свежая сессия-опровержение) → owner-VISIBLE результат → интегрировать + промоутить факты.

- **S0 — ФУНДАМЕНТ + ОЩУЩЕНИЕ (активен, ведёт).** Воксельайзер (generator-blind читатель → грид с гранями + region_id) + face-flow (заполнение комнаты) + §9-швы + debug-гизмо + integer/zero-float scan + loopback-хеш. Netcode-НЕЙТРАЛЬНО. Видимо: уровень→грид, газ ползёт/копится; глаз владельца «весело ли» (1 игрок, малый уровень). Ships: 50см near-грид + face-flow, швы-заглушки, песочница. Defers: всё ниже.
- **S1 — Форсированный поток (СДАНО 2026-06-28, c-exec-015).** Импульс-события + затухающий направленный bias: ВЫБРОС + ВЕТЕР/развилка + односторонний КЛАПАН + jet-emitter (см. §3 «Форсированный поток»). ВЫДАВЛИВАНИЕ (S1b) ВЫРЕЗАНО в бэклог (`d-displacement-s1b-cut-001`). Deps: S0. *(Было: «Выброс + выдавливание» — реформулировано в forced-flow 2026-06-26, s-work-020.)*
- **S2 — Мультиплеер (lockstep) (СДАНО 2026-06-29).** РАЗБИТ на **S2a** (c-exec-017 — ZERO-LEGACY снос pre-lockstep host-broadcast reconstruction-хребта) + **S2b** (c-exec-018 — load-bearing 2-VoxelField loopback: одинаковый по-тиковый MeaningChecksum на 2 ПРОЦЕССАХ, ОДНА машина; целочисленный кросс-CPU = ДАНО; реальные 2 ПК = опциональный одноразовый прогон, НЕ гейт). Детерминизм по построению (integer + zero-float + канонический порядок + сидированный RNG), НЕ через re-flux. ⚠ Доказан LOOPBACK; газ ещё НЕ в реальной сетевой шине инпутов (gas-as-ITickInputBus-citizen = бóльший шов ~S4). Lock = ADR-0002. Deps: S0 (S1 желателен).
- **S3 — Высота / расслоение.** Near: реальная высота (уже эмерджентна из S0). Грубый far-тир получает несколько ВЕРТИКАЛЬНЫХ слоёв для маршрутизации по высоте. ⚠ Эти грубые вертикальные слои — это FAR-тир для комнат БЕЗ игрока, НЕ near-авторитетные «полосы для присесть» (присесть уже бесплатно через 50см Z-клетки в S0). Deps: S0. ⚠ Грубые far-вертикальные страты S3 садятся на **S4-rollup**, НИКОГДА на интерим `CoarseBandStep` (его S4 удаляет — §7/§8). Near-высота эмерджентна (Dep: S0); far-страты Dep: **S4**. Deps: S0 (S3 желателен). ⚠ Пролом ПОЛА/ПОТОЛКА (газ льётся вниз сквозь пробитый пол) требует расширения авторинга проёмов на **Z-нормальные грани** — сегодня `ApertureSpec` = (U,Z) стеновой прямоугольник, `StructureVoxelizer` пропускает Z-нормальные грани; wall-only S5 = осознанный scope-cut, требует owner-ack (не молча). Зависимость: temp-ключевая типизация ждёт отложенной temp→gas обратной связи (`d-tempfeedback-001`) — сокет кладётся, temp-строки СПЯТ; density/соседи могут драйвить раньше. Форки (флип vs аккумулят-с-гистерезисом — owner LEANS аккумулят, НЕ подтверждено; режим vs идентичность) решаются на shape слайса реакций.

## 6. ОТКРЫТО (реально не решено)

1. **`cap` (потолок массы) — дефолт.** Что: опциональный конфиг (`max_mass/rate_limit/dissipation`). Гейтит: ничего жёсткого — rollup несёт полную заливку как поддержанный исход, cap = удаляемая граница. Рекомендация: дефолт OFF, заглушка-поля в чексумме с дня один. Когда: подтвердить на PLAN S4. 2. **Потолок масштаба уровня (число комнат).** Что: целевой ~150 комнат ≈ 223к клеток @50см, 1 «ангар» на уровень (владелец поднял с ~40). NB: ~150 комнат (≈223к) уже выше КОМФОРТНОГО потолка ~200к (≈134 комнаты) на слабом ядре — это и делает S4 room-rollup **обязательным** («опционально» было артефактом 40-комнатного числа). Когда: уточняется hangar-пробой + на PLAN S4. **КОНТРАКТ УРОВНЕ-ДИЗАЙНА:** измеренный потолок ПОЛНОСТЬЮ-детального уровня на слабом ядре = **~35 000 одновременно-активных клеток @50 мс / ~70 000 @100 мс** (слабый пир ×4-прокси; боевой потолок по Step; замер c-exec-023 flooded leg2 = 70.788 мс/196 608 клеток на dev-воркстейшн; посажено s-work-039 2026-07-03, owner «да»). NB: это ОДНОВРЕМЕННО-активный потолок — общее число клеток уровня может быть много больше (cost ∝ активный ФРОНТ, не домен). ⚠ держится на пока НЕзакоммиченном бенчмарке (binding-G5 P2#1) → c-exec-024 делает воспроизводимым. НЕ маркетить масштаб без машинерии; «меньшие уровни» = только с подписью (п.3 фолбэк). [аудит 2026-07-02 global-scale-1 / words-vs-docs-001]

- **25см как авторитетный грид ГАЗА / «25см точечно/покомнатно» / per-tier T0=25см ГАЗ** → **заменено на: ГАЗ 50см глобально авторитетно; 25см в СЛОЕ ГАЗА = ТОЛЬКО off-checksum локальный визуал.** ⚠ Это НЕ запрещает слой СТРУКТУРЫ/РАЗМЕЩЕНИЯ на 25см: у них СВОЁ авторитетное разрешение (Факт 2 ре-шейп 2026-06-26), связанное с газом односторонней целочисленной проекцией (§9 шов 9, conductivity по под-граням). Похоронено именно «25см как разрешение ГАЗА», а не «25см где-либо».
- **re-flux как гейт «на 2 машинах» / «2-machine spike/proof»** → **заменено на: loopback (2 процесса, ОДНА машина) + zero-float scan.** Реальные 2 ПК — НИКОГДА не гейт. re-flux-равенство понижено до owner-eye no-pop, КРОМЕ D5-комнат и ведро-3 (там снова жёсткий гейт).
- **«8 пузырей» (8 игроков = 8 пузырей)** → **заменено на: ~1 пузырь/пир** (далёкий пир пропускает чужой пузырь, он и так считает грубое для тех комнат).
- **Площадь-слепой закон `t=mEq/kP`** → **заменено на: вблизи — поток через открытые грани (count = площадь, R6); на far-тире — area-aware cut-set rollup ТОГО ЖЕ закона (§3).** `t=mEq/kP` = ИНТЕРИМ coarse-транспорт в ТЕКУЩЕМ коде (`CoarseBandStep`); на **S4** заменяется area-aware rollup'ом — НЕ «выживает как far-закон». До S4 живёт как интерим far-тир (как и 2-band, §8).
- **«мгновенный snap к равновесию» (аналитический телепорт к среднему)** → **заменено на: довести фронт до покоя, затем байт-сон замороженного состояния** (collapse, без snap).
- **2.5D-per-tier (как near-авторитет)** → **заменено на: одна 3D-модель near + room-rollup far.** *(NB 2026-06-26: `mid-2.5D` как FAR-only вертикальный LOD дальних НЕВИДИМЫХ комнат = отложенный РЕЗЕРВ под большие залы (Факт 2 / ведущий кадр); включается ТОЛЬКО по замеру, далеко, mass-exact per-layer largest-remainder, НИКОГДА near/при игроке — near остаётся 3D. Не активен, требует owner-подписи на возврат тега.)*
- **Сектор-band граф как far-тир (ARCH-1v2)** → **заменено на: room-graph-rollup far.**
- **Переиспользуемый движок как цель (D12)** → **снято; делаем игру.**
- **«Прорыть коридор в газе руками» (D13)** → **вырезано** (анти-равновесие + отказ back-pressure).

## 8. Сверка с кодом

**Текущий грубый сим = МОДЕЛЬ 2-BAND целиком.** `BandCount=2` (`CoarseSectorGraph.cs:41-43`), Lower/Upper; адресация `sector*BandCount+band`; вертикальная физика = по-полосная плавучесть/заливка (`CoarseBandStep.Settle`, `:183-248`); read-model `CoarseGasReadModel.ConcentrationAt` отдаёт BAND-разрешение (z→полоса через `SubBandInterpolation.BandForHeight`), НЕ 50см-сэмпл; capacity = `FreeVolume/2` (по полполовины комнаты); `ProfileHash` вшивает `BandCount`.

**Это НЕ то, что супер­седит S0.** S0 строит НОВЫЙ 50см near-грид + face-flow как net-new поддерево `Core/Field/Voxel/`, РЯДОМ с грубым тиром (coexistence, не замена). 2-band тир классифицирован **keep-as-far-tier**: его авторитетная роль заменяется на **S4** (room-rollup). До S4 полосы — легитимный far-тир.

- **S0 удаляет (delete-set):** только DA-адаптеры `SnapGridFlowRoomReader.cs` + `SnapGridFlowTopologySource.cs` + reconcile сцены. НОЛЬ файлов грубого сима. Плюс удаляется крутыль `VScale=2` (zero-legacy).
- **S0 сохраняет (keep-set, подтверждён здоровым):** `TopologyDocument` / `TopologyConformance` / `TopologyStableIds` (чистая целочисленная геометрия, без Unity/float — годны для вокселизации 50см); `RoomSpec`/`LinkSpec` (engine-free целочисленные DTO, идентичный вход для DA и ручных уровней); RN1 read-model `IGasReadModel`/`CoarseGasReadModel`/`ExposureQuery` (far/band read-model, frozen, НЕ мутируется — `ConcentrationAt` неприкосновенен; визуал-трек g-7e15 целится в НОВУЮ near-50см read-model «где/сколько», строящуюся РЯДОМ, см. §5); region_id заводится двумя путями (far: новый per-sector массив в `CoarseSectorGraph`, в `ProfileHash`; near: per-cell поле грида).
- **Что S0 ДЕЙСТВИТЕЛЬНО супер­седит:** только DA topology-reader адаптер-путь → generator-blind читатель→воксельайзер.
- **Тиры, заменяющие легаси по слайсам:** авторитетный 2-band CoarseBandStep → S4; по-band температура → `{energy_sum,capacity_sum}` (швы day-one); host+2c reconstruction (LOCKED в коде) → input-lockstep (порт ещё не сделан — это незавершённый хребет миграции).

**Предупреждения для build-сессии:**
- Кодомен collapse в S0 держать абстрактным/per-region; НЕ привязывать S0-collapse к `CoarseBandStore`, как будто 2 полосы — цель свёртки (так полосы и воскресают). В S0 — только тело-тождество.
- Near-чексумм-смысл должен быть отзеркален на follower (`CoarseChunkFollower.ComputeHash`), иначе host≠follower — проверить на build.
- `Conformance` имеет 2 band-связанных проверки (отклонение нечётного `FreeVolume` под `/2`; gate только на `SillBandLowerEdgeHeight`, НЕ на `OpeningSize`) — near-admission добавляется АДДИТИВНО, не ослабляя их.
- В `c-exec-013-call.md:84-86` и `c-exec-013-PLAN-s0.md §G:282` строка cut-list зовёт S3 «authoritative checksummed HEIGHT-BANDS» — это фантом полос. Переформулировать: S3 = более тонкое ВЕРТИКАЛЬНОЕ уточнение (off-checksum near; far-routing-слои), НЕ новый авторитетный раздел-полосы. Правка только в OUT-OF-S0 ярлыке, не трогает ни один S0-deliverable/done_when/RED-тест/код.
- **S0 cross-check = DA-only (owner-approved PLAN R1, 2026-06-23).** Ручные уровни ВЫРЕЗАНЫ; тест «DA-vs-hand байт-идентично» ЗАМЕНЁН на: re-вокселизация-детерминизм (тот же RoomSpec → байт-идентичный грид) + in-code RoomSpec/LinkSpec фикстура. Анти-подмена держится Core-boundary компилятор-гейтом + grep-clean `ActiveModel as SnapGridFlowModel = 0`. «Идентично для DA и ручного» = архитектурный ИНВАРИАНТ, НЕ S0-тест-обязательство — спек/билд НЕ должен требовать hand-теста.

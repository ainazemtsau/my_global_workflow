# Knowledge — Mechanic-design lens set (the gate a game mechanic must pass)

Established: 2026-06-20 (canon-track g-d3a8; owner-signed «да»). Built from 3 research workflows + first-hand repo
reads, after the `canon-forge` process was found to tautologize MECHANICS (it had only a lore-mode terminus — freeze
one atomic invariant card — which compresses a playable loop into a slogan). These six lenses are the MECHANIC gate the
`canon-forge` step-4 craft check was always meant to read; `knowledge/` held no mechanic gate until now.

## How to use

Run a candidate mechanic through the six lenses **strictly in order — cheap paper lenses first, the grey-box last**;
fail cheap before building. A FIX to a mechanic re-enters at lens 1, never in place.

- Lenses **1–5 are PAPER PREDICTIONS**. A clean sweep certifies only "no anti-pattern present + this CAN be
  prototyped" → status **UNVERIFIED**. It does NOT mean "fun".
- **Only lens 6** (the two-player grey-box, run with real humans) certifies a mechanic is **fun** → status
  **CERTIFIED**. Never let a paper sweep masquerade as a fun verdict.
- On a lens-6 fail the answer is **ITERATE-OR-CUT the mechanic — never add a 7th lens** (over-lensing is the
  gate-gamed contrivance we fight). The fewest lenses that each kill a distinct failure; more is a failure.

Frozen core the lenses test against: the **shared, non-local, multi-type gas FIELD** + **not-solo-completable
extraction**; the North-Star feeling **greed↔fear, pulled out TOGETHER**; story is an optional layer (a mechanic needs
no lore reason); pays in loot+depth.

## 1. Grows-From-The-Core   (paper · HEADLINE)

**Catches:** contrivance — a mechanic invented standalone and then justified with backstory ("за уши притянуто"),
instead of falling out of the two core systems. The cure for contrivance is emergence-from-core, not stricter checklists.

**Ask (cheapest sub-check first):**
- **RENAME:** rewrite the mechanic as pure abstract rules, every gas/city/co-op noun stripped. Re-skinnable onto
  medieval / space / farming with ZERO rule change? → the fiction is paint, **FAIL**.
- **DERIVE:** finish "because gas is a shared NON-LOCAL field AND extraction is NOT-solo, THEN this must happen"
  WITHOUT inventing a new rule-primitive. Can't → bolt-on, **FAIL**.
- **DELETE-TEST:** delete the mechanic's bespoke fiction-fact / device — is the emergent situation still reachable
  from gas field + co-op alone? Vanishes → load-bearing scaffolding, **FAIL**.
- **EXPAND-as-limitation (TIE-BREAKER, NOT a gate):** does it expand an existing field property, and does it bite as
  a field-imposed limitation? A limitation-shaped bite is a *stronger* pass — but a genuinely emergent POWER (a scrubber
  carving a temporary safe lane; a vent spent at a shared cost) that still DERIVES and DELETES-clean PASSES. Demoted to a
  tie-breaker so good power-shaped mechanics are not bent into joyless all-stick constraints.

**Fail = ** rename re-skins it / can't derive without a new primitive / deleting the fiction-fact removes the situation.
Any one → back to design; do not advance.

**PASS example —** *Venting*: dumping a built-up gas regime up the shaft to clear your own path RAISES danger for the
partner below in the same shared air. Can't rename (needs shared non-local air); derives with no new primitive; deletes
clean (regime-redistribution is still reachable from gas migration + co-op).

**FAIL example —** *"Resonance crystal"*: a planted device that opens a bonus loot-vault when both players shout into
voice chat at once. Renames to "Magic Gong" with zero rule change (pure paint); can't be derived from gas+co-op (voice
volume + a planted vault are fresh primitives); deleting it removes the situation entirely. It even passes a naive co-op
check (two players DO shout) — which is exactly why this lens runs FIRST: interdependence does not imply emergence.

**Source:** Sanderson's Laws of Magic (1st = the delete-test, 2nd = limitations>powers, 3rd = expand-before-add);
eurogame "pasted-on theme" / rename test (Sarah Shipp, *Thematic Integration in Board Game Design*, 2023); "systems not
content" (Warren Spector & Harvey Smith, immersive-sim roundtable).

## 2. Interesting-Decision   (paper)

**Catches:** a fake or hollow "choice" — a dominant line, or a pick that changes no outcome (complexity, not depth).

**Ask:**
- **TRADEOFF:** name the ≥2 things traded (loot-now vs air-safety; clear-my-path vs endanger-partner; deeper vs costlier
  return). Is giving up the other side genuinely painful? Can't name two → no decision.
- **NO-DOMINANCE:** for EACH option name one concrete LIVE state (gas regime / partner depth / return-route) where it is
  the WRONG pick, and does the best pick FLIP mid-run? An always-right option, or a repeatable always-do-this line a
  competent pair would just run, is dominant → **FAIL**.
- **GREED COST FELT:** does the greedy option carry a cost the PAIR feels (raises shared-air danger / worsens the
  return), not bought off at a cheap flat fee? Fear neutralized by a fixed price → greed↔fear collapsed to pure greed.
- **MOVES-AN-OUTCOME (delete-on-paper):** delete the mechanic — does the reachable set of {live/die, together/alone,
  wealth/poverty} SHRINK? *Conditional:* IF it exposes a continuous axis, that axis must be NON-monotone (more isn't
  simply better); IF it's a binary choice (vent-or-not), the delete-test ALONE suffices — don't demand an axis that
  doesn't exist.

**Fail = ** an always-best option / a repeatable dominant line / a monotone axis (a number to optimize) / greed bought
off cheap / deleting it leaves every outcome still reachable (pure comprehension tax).

**PASS example —** *Regime edge-holding*: hold a volatile cell closer to its edge for higher yield, but the edge bleeds
gas into the SHARED return air. Wrong pick when the partner is deep on the return; right when they're topside and the run
needs cash — best pick flips with partner depth; non-monotone (past the edge the cell cooks off, both die).

**FAIL example —** *"Breathing mask: ignore gas for 30s at a fixed cost"* (dominant — always equip, always push;
fear bought off flat). And *"canisters come in small/med/large"* (monotone — the largest carriable is just better;
deleting the system changes no outcome — pure tracking load).

**Source:** Sid Meier, "Interesting Decisions" (GDC 2012); David Sirlin, depth-not-complexity / dominant moves; Dan
Felder, "Complexity vs. Depth". (The {live/die, together/alone, wealth/poverty} outcome axes are the owner's framing.)

## 3. Anti-Solo Refutation Oracle   (paper)

**Catches:** "solo-with-friends" — a mechanic a skilled duo can split into parallel-solo, violating the LOCKED anti-goal.
Run it as a **refutation**: you, the advocate, must FAIL to solo or parallel-solo it.

**Ask (assume a skilled, communicating duo who mastered it):**
- **COMPLEMENTARY CAPABILITY:** name the asymmetric split — "A does X not Y, B does Y not X, the loop needs both". No
  capability one player lacks at the moment of action → no interdependence, **FAIL**.
- **NON-SEPARABILITY:** write it as two per-player to-do lists. Can both run in parallel with NO live info passing
  between them? Yes → parallel-solo, **FAIL**.
- **LIVE SHARED-STATE COUPLING (ours-specific):** does A's action change B's situation MID-action via the non-local gas
  field, forcing continuous re-adjustment — not a one-time token handoff?
- **BYPASS HUNT:** enumerate cheeses and check each stalls — more TIME, AFK/bot partner, GRIND, and especially
  **SEQUENTIAL ONE-BODY** (do A's half then B's half alone). Any bypass closes it solo → **FAIL**.
- **WEAPONIZED-COUPLING (structural griefing check):** can one player use the shared-air coupling to strand/kill the
  other and still complete alone? Trivially weaponizable into a solo path → **FAIL**.

*Note: "greed raises danger for the other" is NOT gated here — that incentive direction is checked at lens 4. Lens 3
passes on complementary-capability + live-coupling + all-bypasses-stall ALONE, so pure rescue-coupling or non-greedy
synchronized interdependence is not wrongly failed.*

**PASS example —** *Vent-and-regulate*: A holds a vent open at the SOURCE while B works the REGULATOR downstream; venting
pushes a toxic plume through the shared air toward B, so B must throttle to A's live vent rate and A must modulate to
B's depth-gated readout (which only B sees). AFK partner → the plume kills the vent operator; sequential one-body fails
because the danger is continuous, not set-and-leave.

**FAIL example —** *"Two-valve seal"*: each player carries a canister to one of two valves; both open → the next shaft
unseals. Identical actions (no complementary capability), two parallel to-do lists, no live coupling, and one player can
open valve A then walk to valve B alone (sequential one-body). Textbook solo-with-friends.

**Source:** Depping & Mandryk, "Cooperation and Interdependence" (CHI PLAY 2017) — interdependence (complementary
mechanics, neither succeeds alone) vs mere cooperation. Live-coupling clause = ours-specific tightening, grounded in
`g9c41-wave1-consistency-not-depth-not-coop.md`.

## 4. Pillar-Pull / No-Dissonance   (paper)

**Catches:** wrong-pull and ludonarrative dissonance — a mechanic that grows-from-core and offers a real choice still
fails if optimal play pulls toward a forbidden feeling, or the PAYOUT contradicts the value its fiction sells, or greed
can be weaponized into a profitable grief.

**Ask (on paper, before writing rules):**
- **NAME THE PILLAR:** name the ONE frozen North-Star pillar this is for (greed↔fear-together / "pulled it out TOGETHER"
  / shared-air interdependence) — one, not a list. Can't name the feeling before the rules → aesthetic bolted on, **FAIL**.
- **DIRECTION:** under optimal play by a competent pair, does the felt pull move toward greed-vs-fear-together, or toward
  a FORBIDDEN aesthetic (solo-completable / parallel-solo / dread-without-greed / safe-turtle)? Forbidden → **FAIL**.
- **REWARD-AGREES:** trace the greedy act's payout vector — does shared-air greed pay the ACTOR (faster/richer/deeper)
  AND raise danger in the SHARED air for others? Payout that punishes the greed the fiction sells, or pays strictly
  per-player while the fiction claims "shared", is dissonance → **FAIL**.
- **DELETE-THE-VALUE:** swap the stated feeling for its opposite — must any rule or number change? Nothing changes → the
  feeling is decoration, **FAIL**. (Distinct from lens 1: lens 1 deletes a FICTION FACT "still ours?"; this deletes the
  FEELING "would the payout notice?".)
- **WEAPONIZED-GREED:** is the incentive-optimal line to use shared-air greed to kill/strand the partner for their loot?
  If betraying the partner pays, the reward pulls against "together" → **FAIL**.

**PASS example —** *Venting your loot-rich pocket* lowers YOUR exposure but dumps gas into the SHARED return shaft,
raising danger for whoever is below. Pays the actor AND endangers the partner (payout vector == fiction value); flip the
feeling to "safe solo extraction" and the shared-shaft rule must change.

**FAIL example —** *"Stabilizer minigame"*: each player solos a QTE at their OWN console, scored PER-PLAYER, while the
fiction claims a "shared regime". Optimal play is parallel-solo; per-player payout contradicts the shared fiction (same
shape as BioShock's harvest-pays-more-than-rescue); flipping the feeling changes nothing — pure decoration.

**Source:** MDA (Hunicke, LeBlanc & Zubek, 2004) — design backward from the aesthetic; Clint Hocking, "Ludonarrative
Dissonance in BioShock" (2007).

## 5. Field-Read & Skill-Atom   (paper)

**Catches:** a real sim decision that is dead in play — invisible, unlearnable, or unfair when it fails.

**Ask:**
- **FIELD-READ:** (1) name the ONE state-change the player must read to decide ("this air just went lethal"; "the front
  is rolling toward our exit"). (2) Is it read through a NON-numeric perceivable channel (colour / fog density / sound /
  a visibly moving front / a diegetic gauge), NOT a HUD value? "A number in the corner / on the cassette UI" → automatic
  **FAIL**. (3) Can a BYSTANDING teammate read the same state in their own space (shared air)? (4) Does the read arrive
  BEFORE the consequence is irreversible, with time to choose?
- **SKILL-ATOM:** (5) write the one-sentence atom (action → sim changes → player SEES feedback → infers the rule).
  (6) Can a never-told player infer it by acting+watching once in the first minute, no text wall? (7) Can a teammate
  teach the whole rule in one spoken sentence ("hold the valve till it stops glowing")? (8) Is a wrong first attempt
  informative + recoverable, not opaque punishment?
- **CATASTROPHE-READ:** (9) when this mechanic goes WRONG, can the pair diagnose WHY (the lethal consequence is
  attributable, not a random death) AND is recovery reachable, not an unrecoverable spiral?

**Fail = ** deciding state knowable only via a HUD number / post-hoc death; a bystander can't read a consequence dumped
into the shared air; understanding needs a tutorial wall or one-driver-one-spectator; a wrong first try is opaque; or
failure reads as random / spirals unrecoverably.

**PASS example —** *Breach/vent route-making*: opening a remote bypass sends a VISIBLE gas front (colour shift + rising
density + audible pressure) rolling toward a teammate's room over a few seconds; both actor and bystander read it in
time. Atom: "hold the quench valve till it stops glowing" — one sentence, learned in one action.

**FAIL example —** *"Gas-purity payout"* shown only as a NUMBER on the cassette UI after pickup (un-readable in the
field → collapses to "always wait for the number"); and a *"metagas charge"* meta-layer built from a hidden composition
ratio across rooms (no single action shows anything; a cold co-op player can't infer it; lives in a wiki).

**Source:** Jesse Schell, *The Art of Game Design* (Lens of Indirect Control / Visual Design); Sanderson's First Law
(legibility); Daniel Cook, "The Chemistry of Game Design" (skill atoms).

## 6. Grey-Box Two-Player FUN-Test   (grey-box · UN-FAKEABLE · BINDING · LAST)

**Catches:** everything paper can't — a mechanic that survives all five paper lenses yet two real humans don't FEEL.
This is the ONLY lens whose evidence is observed live behaviour, and the ONLY one that certifies fun.

**Ask:** actually RUN the cheapest grey-box of ONLY this mechanic (placeholder cubes, the REAL gas sim, no
art/story/economy) with TWO real humans, at least one COLD. No run → status UNVERIFIED, never PASSED; stop. Judge
OBSERVED behaviour, not self-report (Schell's six playtest questions):
1. **GREED↔FEAR unprompted:** do BOTH visibly show the tension (lean in, hesitate, "one more / let's go")? One engaged +
   one bored → **FAIL**.
2. **INTERDEPENDENCE not just cooperation:** did a moment REQUIRE two people (rescue, shouted warning as the front rolls
   in, "you hold the valve I grab it", one's greed dumping into the OTHER's air)?
3. **PARALLEL-SOLO CONTROL (load-bearing):** could one competent player split the same actions across two characters with
   no real-time reliance and get the same result? YES → cooperation-flavoured solo (locked anti-goal), **FAIL** even if
   it looked fun.
4. **ARTICULABLE SIGNAL:** can each player, unprompted, say how a specific gas behaviour changed a specific decision
   (distinct from generic "hazard/noise")?
5. **ATTRIBUTABLE DANGER:** was a partner's greed-raised danger readable/attributable, not random?
6. **REPLAY PULL:** did they spontaneously want to go again?
- **COMPOSITE-LEGIBILITY:** run it at least once ALONGSIDE one already-certified mechanic — Sim-Tower noise emerges when
  mechanics collide; check the gas doesn't collapse into ambient noise when two fire at once.
- **ANTI-SIM-TOWER:** if the only "fun" evidence is "the sim behaves correctly / is bit-exact", discard it and judge
  Q1–Q6 from behaviour.

**Fail = ** players bored/confused; only one engages; the parallel-solo control reproduces the result; gas reads as
ambient noise they can't articulate/attribute; danger reads as random; no replay pull; or "fun" asserted only because
the sim runs. On FAIL → ITERATE-OR-CUT, never add a stricter lens. Only a RUN grey-box flips status to CERTIFIED; paper
may never pre-empt this verdict.

**PASS example —** *Containment/quench shared-carry*: a cassette BOTH must hold; two players (one cold) hit an unprompted
"grab the deeper one or bolt" argument as the front rolls in, shout warnings, one rescues the other, barely escape, say
"again". The parallel-solo control fails — one body can't carry AND cover at once.

**FAIL example —** *Sim-Tower*: the multi-type gas field itself, paraded as a mechanic. It propagates beautifully and is
bit-exact host+clients (proven g-9c41 Wave 1) and passes every paper lens — but in the grey-box the gas reads as ambient
hazard, players can't say how a gas TYPE changed a decision, one competent player clears the room (parallel-solo), danger
reads as random. The only "fun" claim is "the sim runs". → ITERATE-OR-CUT.

**Source:** Jesse Schell, *The Art of Game Design* (Lens of the Playtester + the six questions) — only real play answers
"is it fun?"; Will Wright, SimCity "software toy" (a correct simulation is not yet a game) → the anti-Sim-Tower rule;
Depping & Mandryk (the parallel-solo control). Internal: `g9c41-wave1-consistency-not-depth-not-coop.md`.

## Run order (fail cheap first)

**1 Grows-from-core → 2 Interesting-decision → 3 Anti-solo → 4 Pillar-pull → 5 Field-read → 6 Grey-box (binding).**
The three lenses that touch the solo anti-goal sit at different altitudes and reinforce, never collide: **3** = solo is
structurally impossible (paper refutation); **4** = solo is un-rewarding (incentive vector); **6** = two real humans
actually needed each other (empirical control).

## What we deliberately DO NOT gate on (anti over-lensing)

- **Novelty / originality** — an OUTPUT of emergence-from-core (lens 1), not an input requirement; gating it breeds gimmickry.
- **Technical feasibility / netcode / performance / buildability** — the engineering contour, not a design lens. (Honest
  hole: the set has no cheap paper signal for "passes every lens yet needs 6 months of bespoke tech"; that only surfaces
  at the grey-box — accepted, rather than add a buildability lens that bends design toward cheap-to-build.)
- **Lore / story richness** — story is an optional layer; only fiction CONTRADICTION is gated (lens 4), never quantity.
- **Commercial / clip-ability** — the CHARTER commercial lens + marketing track.
- **Realism / sim-fidelity** — anti-Sim-Tower; never a pass condition.
- **Exact tuning / balance numbers** — iterated AFTER a mechanic earns its place.
- **Accessibility (colour-blind / audio-only)** — real, but a production-stage concern, not a design failure.

## Honest limit (the direct answer to "everything feels за уши")

Lenses 1–5 (paper) remove the "за уши" feeling BEFORE building → status **UNVERIFIED** ("not contrived, worth a
prototype"). Only lens 6 (a live two-player grey-box) proves "fun" → status **CERTIFIED**. Paper never certifies fun;
don't let a clean paper sweep masquerade as a fun verdict.

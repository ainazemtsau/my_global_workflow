# Knowledge — g-9c41 Wave 1: consistency is NOT depth and NOT co-op-interdependence; both are unowned on the core

Established: 2026-06-16, review c-review-001 (core-depth + co-op-first lens harvest).

Wave 1 proved **networked consistency** (host + 2 clients see the same gas world bit-exact, even under
drop/delay/reorder, across a real controlled breach). That is necessary but **not sufficient** for either the
core-depth lens or the co-op-first lens:

- **Depth (criterion 10):** the cross-layer coupling is a pure **inert one-way temperature sink** (reaction →
  temperature, fixed `ReactionHeat=32`, no feedback). "Observable cross-layer interaction" currently lets
  *transport* pass as *depth*. The extensible seam is **argued structurally but never exercised** (no actual
  third layer plugged in during Wave 1).
- **Co-op (pre-mortem #5):** does the shared gas state **CREATE** dependency / communication / rescue / shared
  planning, or just **render** it consistently? Unproven. No node owns this before a **parked Steam Playtest**
  (g-d3a8 done_when re-homes "co-op feels meaningful" to that playtest) — late for a co-op-**first** game.

**Consequence:** the Wave-2 shape should decide whether the next wave **owns** a depth-FEEDBACK cross-layer
interaction (not a sink) and **exercises** the seam with a real added layer (the criterion-10 tightening folded
into c-shape-wave2). A "co-op interdependence proof" outcome node on the real core is proposed to map session
c-map-003.

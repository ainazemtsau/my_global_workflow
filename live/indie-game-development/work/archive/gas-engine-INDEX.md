## ГДЕ МЫ (один абзац)

Архитектура газа разобрана до твёрдой точки. **База = «честный Variant-A»:** одна общая ГРУБАЯ
целочисленная правда (сколько газа в комнате/под-секторе, на какой высоте, куда течёт) — каждый пир
считает сам, бит-в-бит (input-lockstep); мелкое поле 25–50 см — у каждого игрока, и оно же его бьёт
(visual==contact); тяжёлые последствия = события из грубого; устоявшийся газ спит (byte-sleep), считается
только активный фронт. «Революционные» альтернативы проверены и **не бьют базу**: грубо-авторитет+GPU
(B) — кладёт обман на границу контакта, ты это уже отверг; целочисленный LBM-ядро (C) — ноль shipped-игр,
рвёт кэш в ангаре. Они остаются **только косметикой**. **Следующий конкретный шаг — измерить ангар**
(игрок в 256k клеток на слабом одном ядре) — это единственное неизмеренное число всей затеи.

## ЖЁСТКИЕ ОГРАНИЧЕНИЯ (обойти нельзя — это рамка)

1. **Только целые в авторитетном пути.** Float туда = рассинхрон между процессорами. (Гейт: zero-float scan, одна машина. Две машины НЕ нужны — `d-reflux-gate-001`.)
2. **Lockstep: каждый пир пересчитывает ВСЁ авторитетное САМ.** Распределить работу по ПИРАМ нельзя (сломает «только инпуты»). НО **на своей машине можно параллелить по ЯДРАМ детерминированно**: газ-сетка = чистая функция соседей прошлого тика (двойная буферизация) → порядок потоков не важен → бит-точно и между ядрами, и между машинами (Burst/Jobs, целые, без atomics, суммы для чек-суммы — в каноническом порядке). → ×4–8. **Потолок = активные_клетки / число ядер слабейшего ноута** (НЕ «одно ядро»). GPU-как-авторитет — нельзя (недетерминирован между видеокартами), только косметика.
3. **Контакт == картинка у игрока.** «Я в газе / урон / лут» читает авторитетную клетку. Обман (иллюзия) разрешён ТОЛЬКО косметически, внутри авторитетного фронта — НИКОГДА на границе контакта.
4. **Стена — игрок в большом открытом зале (ангар, ~256k клеток).** Это рядовой случай (просто стоишь там), и он НЕ ИЗМЕРЕН. Реальный потолок — кэш/память, не флопы.
5. **Активный фронт = реактивность** (считаем только изменившееся; реакции в КАНОНИЧЕСКОМ порядке, иначе десинк) — экономит только пустые/устоявшиеся места. Каскад (огонь метёт сеть залов), вент/насос (держит комнаты бодрыми вечно) и 8 манипуляторов — возвращают полную стену. Многоядерность даёт постоянный ×N, но НЕ меняет переменную (активные клетки) → кит-пробу ангара мерить СРАЗУ на всех ядрах.

## КОМПРОМИССЫ И РЫЧАГИ (где решать — и где можно ХИТРО ОБЫГРАТЬ геймплеем)

| Развилка | Дёшево | Дорого | Где «обыграть» |
|---|---|---|---|
| **A/B по механике** | комнато-гранулярно (высота/лужа/слой/переток) | под-комнатная ФОРМА с общим последствием (струя решает кого накрыло, туннель кидает газ дальнему) | большинство механик — A; форму сделать **косметической** ИЛИ **промоутить в грубое** ИЛИ платить ведро-3 в редких комнатах |
| **Размер клетки** | 100 см (дёшево, грубо) | 25 см (бескомпромиссно, дорого) | ⚠️SUPERSEDED→§12: газ **50 см ГЛОБАЛЬНО**; 25 см = опц. локальный визуал ВНЕ чексуммы (НЕ покомнатный размер); геометрия = целое кратное газа (≥50 см) |
| **Масштаб уровня** | ~40 комнат, всё мелко, −40% инженерии | ~300 комнат + вся LOD-машинерия | fallback, если ангар не лезет: маленькие уровни, **глубина у игрока сохраняется** |
| **Активная манипуляция держит комнаты бодрыми** | — | вечный счёт | ⭐ превратить в **геймплейную НАПРЯЖЁННОСТЬ/ресурс**: нельзя держать весь уровень «взбаламученным» |
| **Большие открытые залы — худший случай** | — | дорого | ⭐ **дизайнить уровни так, чтобы залы были РЕДКИ/фирменны**, а не нормой |

⭐ = ровно те места, где **ограничение движка может стать фишкой геймплея** (то, что ты хочешь обсудить).

**Параллелизм/реактивность (РЫЧАГ, не компромисс):** активные клетки раскладываются по ЯДРАМ слабого ноута
детерминированно (Burst/Jobs, целые, двойная буферизация, без atomics) → ×4–8 по стене ангара; реактивность =
наш активный фронт (реакции в каноническом порядке). ⚠ Проверить отдельно: реально ли Burst бит-точен между
РАЗНЫМИ машинами (нет порядко-зависимых сумм; разное число ядер не влияет) + как резать ангар по ядрам.

**Про модель Factorio / «слать события вместо инпутов»:** мы УЖЕ как Factorio — по сети только ИНПУТЫ (D1),
каждый считает сам. Это не убирает стену, это её ИСТОЧНИК (как UPS-падение мегабазы бьёт всех). Слать по сети
СОБЫТИЯ/состояние = вернуться к bandwidth-стене (события ∝ движущемуся газу = огромны; Factorio шлёт инпуты
именно поэтому) + одному авторитету + потере бесплатной миграции хоста. Инпуты ✓, события на провод ✗.

## УЖЕ РЕШЕНО В ЭТОЙ СЕРИИ (зафиксировано / рекомендовано)

- **Неткод = input-lockstep** (D1 / ADR-0010). Зафиксировано.
- **Детерминизм** (`d-reflux-gate-001`, зафиксировано): integer = данность; гейт = zero-float scan + loopback ОДНА машина; реальные 2 ПК — никогда не гейт.
- **Модель контакта** (рекомендовано): урон/лут/визуал читают ОДНУ `expand(грубого)` на одном разрешении → совпадают by construction; косметика только внутри фронта; засосать/поджечь/сменить состояние = детерминированная запись в грубое; урон дальнего друга считают ВСЕ из грубого (по сети ничего не «передаётся»).
- **База = Variant-A** (рекомендовано, не залочено): грубая целая правда + мелкий контакт у игрока + активный фронт + lossless byte-sleep. **B → только косметический GPU-слой. C → отложенная косметическая momentum-полоса (струи/вихри, что ВИДНО, но не решают контакт).**
- **Первый шаг = проба ангара** до сети/сна/GPU.

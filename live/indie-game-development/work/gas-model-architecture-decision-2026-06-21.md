## 0. TL;DR — конечная архитектура одной фразой

**Одна модель — целочисленные клетки газа + таблица химии — разного РАЗМЕРА клетки по расстоянию.** Вблизи —
мелкие 3D-клетки (полная мощь: облачка, обтекание, воздух). Дальше — клетки крупнее. На самом краю — одна клетка =
комната («ведро»), но ТОЛЬКО для мелких/далёких комнат; большие/важные комнаты держатся пространственно. Один
источник правды; **все механики работают везде**, точность падает с расстоянием. Сеть — **input-lockstep** (по сети
только нажатия). Всё авторитетное — целые числа + checksum.

## 2. Почему именно так — ключевая разблокировка

**Провод ФОРСИЛ грубость.** В модели «хост считает → рассылает картинку» нельзя переслать мелкие клетки на весь
уровень — формат упирался в потолок (255 чанков × 32 = 8160 ячеек/план; + подтверждённый тихий баг `ChunkEncoder.cs:125`
`(byte)c` без гварда → за 255 чанков заворачивает молча, хеш зелёный). Поэтому дальнее ОБЯЗАНО было схлопываться.

**Input-lockstep убирает провод как ограничение** (по сети — только нажатия; сложность газа сеть не трогает). Тогда
единственный предел — **процессор каждого ПК** (по слабейшему пиру). При маленьком уровне это позволяет держать
**мелкие 3D-клетки там, где надо**, и грубее — где не видно. LOD становится **чисто локальной** оптимизацией
(каждый пир решает одинаково, детерминированно) — без провод-боли (нет «попа» на проводе, нет meaning-hash для
реконструкции, нет страниц). Доказано в коде: `SimCore` — рабочий детерминированный редьюсер input-lockstep
(пиннинг `(PeerId,Command)`, `unchecked` целые, seeded RNG, `Hash`).

## 3. Модель в деталях

### 3.1 Данные
- Мир = граф **комнат** (выпуклых кусков) + **проёмов** (рёбер). Газ течёт между комнатами только по рёбрам.
- **Клетка** хранит: доминирующий тип газа + количество (целое) + (на реакции) временный оверлей смеси;
  по вертикали — слои (3D-воксели вблизи / колонка высоты в 2.5D / слои-на-комнату в ведре).
- Всё авторитетное — целые/фикс-поинт. Каждый тик — checksum; рассинхрон → громкий стоп.
- **Сохранять**: масса (целая), температура как пара `{energy_sum, capacity_sum}` (НЕ предделёное среднее — иначе
  схлопывание не идемпотентно), flux-остатки на рёбрах.

### 3.2 Тиры (LOD = размер клетки; один источник правды)

| Тир | Где | Клетка | Что «знает» | Цена |
|---|---|---|---|---|
| **T0 вблизи** | комната игрока + соседние | 3D, ~25 см | всё: форма, высота, облачка, угол | основная (пузырь мал) |
| **T1 средне** | докуда дойдёт за ~минуту (≈3–5 комнат) | 2.5D / крупные клетки (0.5–1 м) | где в комнате газ (угол), слои — примерно | малая |
| **T2 далеко** | дальше | несколько клеток на комнату | грубо где + сколько | крошечная |
| **T3 очень далеко** | край | 1 клетка = комната («ведро») | только «комната X% полна, газ A/B, течёт в соседнюю» | ~ноль |

- **Тир = f(расстояние, размер/важность комнаты).** D5: большие/важные комнаты НЕ уходят в T3 (1 клетка) — они
  остаются пространственными (просто крупнее клетка + реже тик), чтобы возврат был непрерывным (без «перескока»).
- **Переходы (collapse/expand) — единственный путь конверсии**, чистые целочисленные функции, масса точна:
  - `collapse()` (засыпание): сложить массы клеток в крупную клетку (точно, без округления).
  - `expand()` (пробуждение): раздать массу крупной клетки по мелким фикс-правилом largest-remainder → сумма сходится
    до бита (нет «попа» массы). На возврате **форма** грубых тиров регенерируется; для T0/T1/важных комнат состояние
    держится, так что игрок не видит «перескок» (D5).
  - **Re-fluxing** (анти-«поп»): поток через дверь на грубой стороне = целочисленная СУММА мелких потоков → агрегат
    через проём бит-идентичен независимо от тира соседа.

### 3.3 Детерминизм
- input-lockstep + целые + seeded RNG + фикс-порядок (пиннинг по id) + checksum.
- LOD/сон/пробуждение/переключение тира — **чистые целочисленные функции состояния** → все пиры решают одинаково.
- checksum покрывает: массы + температуру + flux-регистры + теги представления/разрешения + **канонический** дайджест
  очереди-пробуждений/активного-набора (сорт по tick, region_id, species-id — НИКОГДА порядок контейнера).
- Миграция хоста — почти бесплатна (у всех есть состояние) → снимает «умер хост = смерть сессии».

### 3.4 Взаимодействия игрока = команды
- Каждое действие (поставить/бросить источник, открыть/закрыть вент, поджечь, поставить отсос) = целочисленная
  КОМАНДА на плоскости ввода, повторяется идентично у всех (механизм как у уже работающего пролома).
- Силовые (ветер толкает игрока, всасывание, декомпрессия) — нужен будущий шов: `FluxAcross(opening,rev)` (целый
  поток, уже считается) + ОТДЕЛЬНЫЙ хеш кинематики игроков (хеш газа их не ловит). Заложить как швы, не строить сейчас.

### 3.5 Химия (где основная глубина)
- Целочисленная **таблица столкновений**: типов газа — каталог ∞ (данные); одновременно ~16/уровень (макс ~32,
  настраиваемо); на комнату ~4, перелив → **очередь реакций** (не тихий сброс).
- Реакции: слияние / аннигиляция / превращение в новый тип; ультра-быстрый газ; проход сквозь стены по типу;
  поглощение звука/света — всё ДАННЫЕ в таблице/пер-тип-параметрах.
- Вблизи: видимое смешивание + **телеграф** (мигание «вот-вот рванёт») + **окно подготовки** (не мгновенно).
  Решение «рвануть» читает **рефайнмент-инвариантные тоталы** (грубые), не пост-распределённые мелкие клетки —
  иначе подход игрока сам детонирует облако (геймплейный «поп»). Цепь — следующим тиком (каскад как ползущий фронт).

### 3.6 Высота / расслоение
- Целочисленная сортировка слоёв (densest → пол; температура сдвигает эффективную плотность → пересорт, со стабильным
  tie-break). Лёгкий — потолок, тяжёлый — пол; несколько газов = разные слои; «лёгкий заперт под потолком» = газ в
  верхнем слое. Работает одинаково на всех тирах (T0 3D точно, ниже — грубее), поэтому переход бесшовный.
- **2.5D предел:** не умеет оторванный 3D-сгусток в воздухе с пустотой и сверху, и снизу (дымовое кольцо) — это только
  T0 3D. Для хоррора почти всегда неважно.

### 3.7 Разрушаемость
- Свободно по клеткам ВНУТРИ комнаты (граф не меняется) — где угодно, любого размера.
- Пробой МЕЖДУ комнатами (новое ребро) = должен быть **пред-объявлен** на генерации (спящий проём), активируется при
  разрушении. **«Плотное меню»** = таких точек много (через 25 см–1 м) → ощущается почти свободно. Площадь/позиция
  пролома влияют на поток (надо дописать площадь в закон).
- Инвариант: НИКАКОЙ пере-разбивки геометрии в рантайме. Обвал-делящий-комнату (Case B) = линии разлома пред-нарезаны
  на генерации.

### 3.8 Ветер / температура (позже, но везде)
- Ветер/вент = целочисленный модификатор потока на ребре (НЕ float-солвер). Каверза: насос не даёт региону «заснуть»
  (вечный поток) → бюджетировать как постоянно-горячее ребро; нужен named-сток, чтобы оставаться консервативным.
- Температура — независимый слой (пер-клетка вблизи, пер-регион далеко); влияет на плавучесть. Пер-видовая температура
  (горячий тяжёлый всплывает) — будущий шов (ломает «без-сортировки» оптимизацию + нужны планы; заложить запас адресов).

## 5. Доказательство: все механики работают на всех тирах

«Работает везде» = те же правила на клетках разного размера; крупнее клетка → больше погрешность, но механика не выключается.

| Механика | Вблизи 3D (T0) | Средне (T1) | Далеко/ведро (T2-T3) | Везде? | Зависит от |
|---|---|---|---|---|---|
| Распространение/налив/слив | точно 3D | крупнее | по фронту комнат | ✅ | целочисл. перенос + flux-register |
| Высота/расслоение | точно (слои/воздух) | колонки 2.5D | слои на комнату | ✅ | целочисл. сортировка слоёв |
| Площадь → сила потока | точно | точно | по площади двери | ✅ | дописать площадь в закон |
| Реакции двух газов | линия контакта + телеграф | примерно где + телеграф | «A и B в комнате → реакция», очередь | ✅ грубее далеко | таблица химии |
| Цепные/каскад | по-клеточно, след. тик | крупнее | на уровне комнат | ✅ | распространение события |
| Разрушаемость | свободно по клеткам | крупнее | флип рёбер | ✅ | рёбра + (вблизи) клетки-стены |
| Ветер/потоки | толкает клетки | смещает поток | перекос в двери | ✅ | модификатор ребра + сток |
| Температура | по клеткам | по region | по комнате | ✅ | целочисл. аккумулятор тепла |
| Проходимость/укрытие | точно где газ | примерно | «загазована да/нет» | ✅ грубее далеко | read-model |
| Лут «газ коснулся» | точная клетка | дошёл до угла (примерно) | %+задержка (только T3) | ✅ (T3 по %, разрешено) | доза-накопитель + wake-on-resident |
| Облачка/разрыв на препятствии | да (витрина) | частично | нет (один сгусток) | ✅ где важно (вблизи) | 3D вблизи |
| Много типов газа | да | да | да | ✅ | разреженно + таблица химии |

**Честно НЕ делает (и почему ок):** точное «где» реакции далеко (на уровне комнаты; точное — позже граф-фронты);
форма облака на возврате грубых тиров перемешивается (только очень далеко; важные комнаты держим — D5); висящий 3D-сгусток
только вблизи.

## 6. Что облако-математика дала (финальный ресёрч)

- **Авторитетное «облако из чисел» вдали = «ведро»** (комната = слои×типы целыми + поток по дверям). Это и есть честный
  ответ «опиши газ числами + эволюцию»: целочисленно, дёшево, ложится в checksum, цена ∝ числу комнат/фронтов, не объёму.
- **Плавный гауссов-блоб/метабол** (Gaussian-puff/moment-метод) — красивый и правильный МЕНТАЛЬНО, но **в помещении только
  ВИЗУАЛ**: стены дают комбинаторные отражения; `exp/erf/sqrt` не бит-идентичны на разных CPU; обрезанный гаусс уходит в
  МИНУС (прошлая проба s-arch-003: минус на 31% ширины комнаты) → ложные взрывы. Рисуем блобом — **храним/сверяем ведро**.

| Что | Почему отвергнуто |
|---|---|

## 8. Отложено / будущие оптимизации (с триггерами — НЕ сейчас)

| Оптимизация | Когда | Профит |
|---|---|---|
| **Активный фронт / dirty-set** (считать только изменившиеся комнаты + соседей) | **ДЕНЬ ПЕРВЫЙ** (это главный рычаг, не отложка) | ОГРОМНЫЙ — полуровня спящего газа почти бесплатна |
| Слияние насыщенных цепочек + pressure-teleport | по мере роста | БОЛЬШОЙ |
| Меньшая герцовка вдали (степень двойки + целочисл. перенос остатка) | по мере роста | средний (НЕ настоящий variable-rate — теряет массу) |
| **Предсказательный таймер пробуждения (QSS)** | ПАРК до ~10× масштаба + телеметрия длинных тихих окон | средний (сейчас дежурная цена доли мс — переоценён) |
| **Граф-фронты** (точное «где» реакции далеко: реакция в узле встречи фронтов) | Wave-3, после полдня-спайка на 2 машины | закрывает уступленный дальний пробел |
| Разреженный хеш-склад (VDB/hashed) | только если плотный «доминирующий» массив не хватит (при насыщении плотный выигрывает) | переменный |
| ILGA (Integer Lattice Gas) — богатое бит-точное движение | локальный прототип, если «вода» бедна | качество движения |
| GPU-авторитет (целый Margolus block-CA, CRC vs CPU) | только если масштаб потребует; ресёрч-grade | перф (с ~20-30% налогом) |
| Пейджинг / мульти-неймспейс провода | только если когда-нибудь вернёмся к host-broadcast | масштаб |

## 9. Швы, которые ЗАЛОЖИТЬ СЕЙЧАС (чтобы дальше — довески, не переписывание)

1. **Площадь проёма** в законе потока (сегодня `OpeningSize` есть, но игнорируется; `t=mEq/kP` без площади).
2. **`representation_tag` + `resolution_tag` на регион** {fine-3D | mid-2.5D | coarse | bucket} — переключаемо.
3. **Целочисленный flux-register (остаток/перенос) на КАЖДОМ ребре** с дня первого (даже fine-to-fine) — делает все
   будущие швы coarse/fine и сон/пробуждение без потерь массы.
4. **`collapse()/expand()` — единственный путь конверсии**, чистые целочисленные, по слоям (никогда не усреднять колонку
   в один скаляр), привязаны к инварианту остатка.
5. **checksum покрывает смысл, не только значение**: TypeId слота, живой размер клетки/тир, page-id, канонический порядок
   очереди-пробуждений. (Иначе тот же класс тихой порчи, что подтверждён в `ChunkEncoder`.)
6. **Температура = `{energy_sum, capacity_sum}`** (не среднее) — идемпотентное схлопывание.
7. **`FluxAcross(opening,rev)`** (рефайнмент-инвариантный целый поток) + **отдельный хеш кинематики игроков** — для
   будущих силовых механик; решить ДО первой силовой.
8. **Гард `(byte)c` в `ChunkEncoder`** — починить молчаливый оверфлоу сейчас (даже в lockstep пригодится для снимков).
9. **Атрибуты ребра** (направление-маска для односторонних клапанов, целая проводимость для частичного блока, порог
   взрыва) — зарезервировать.
10. **Запас адресов планов** (если когда-то понадобятся доп. реплицируемые слои — фронт, пер-видовая температура).

## 11. Риск-первый план де-риска (headless; что доказать ДО законов)

1. **🔑 Грей-бокс ОЩУЩЕНИЯ вблизи в 3D** (первый и дешёвый): газ ползёт/копится/сливается, одна реакция телеграфит+бахает,
   вент тянет — на реальной сетке. «Весело ли?» — owner-eye. Главный make-or-break (нельзя юнит-тестировать).
2. **spikes-before-faith (по полдня, на 2 машины, бит-идентично):** активный-фронт; слияние-сегментов; пробуждение —
   каждую заимствованную оптимизацию доказать бит-идентичной на 2 машинах ПЕРЕД верой (их шипили по отдельности, но не
   вместе с бит-точным lockstep + переключением представления).
3. **Ключевой оператор потока** (один трансфер на проём, площадь+высота, carry, потом разнос) + re-fluxing-инвариант на
   границе coarse↔fine (агрегат бит-идентичен).
4. **collapse/expand** бит-точно: масса сходится, схлопывание идемпотентно, форма важных комнат держится (D5/P9).
5. **Щель-сочится** (carry): мельчайший проём >0, не запечатывается в ноль.
6. **Гард провода** + масштаб 300 комнат: либо лезет, либо падает ГРОМКО, не молча.
7. **checksum смысла**: красный негативный контроль на каждое измерение (TypeId/тир/page/порядок очереди) — мутируем поле
   при тех же значениях → хеш ОБЯЗАН измениться.

## 13. Открытое / НЕ обсуждённое (флагнуть, не потерять)

- Точные **радиусы тиров** + **размеры клеток** на тир (25 см / 0.5 / 1 м) — тюнинг на дизайне/пробе.
- Точная политика **«большая/важная комната = авторитет»** (порог размера, как держим её дёшево: крупнее клетка + реже
  тик, но пространственно) — D5 принят как принцип, детали открыты.
- **Возврат-издалека**: владелец склонился к «держать состояние» (ближе к варианту «место авторитетно»); для совсем
  дальних мелких — перемешивание формы ок. Точную границу решить.
- **Силовые механики** (ветер толкает игрока) — швы §9.7 заложить ДО первой такой механики.
- **Смена фаз** (R19) — не запрещать, спецификация потом.
- **Дальние оптимизации** в деталях (QSS-горизонты, пейджинг, GPU) — владелец оставил «на потом».
- **3D-near стоимость на больших ангарах** — 3D 25 см в большом открытом объёме дорог (ангар ~256k клеток); вблизи это
  пузырь игрока (ок), но если игрок В ангаре — нужен бюджет/деградация; проверить пробой.

## 14. Сверено по живому коду (ground truth, на чём держатся выводы)

- Поток `t=mEq/kP` БЕЗ площади (`CoarseBandStep.cs`); `TopologyPortalSpec.OpeningSize` есть, но игнорируется.

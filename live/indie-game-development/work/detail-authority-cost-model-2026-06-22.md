## 1. Вердикт: НЕТ, 8 пузырей считать не надо — интуиция владельца ВЕРНА (~8× экономия), но УСЛОВНО

Каждый пир считает **дешёвую ГРУБУЮ симуляцию (на уровне комнат) для всей карты** + **дорогой ДЕТАЛЬНЫЙ пузырь только там,
где его собственный игрок.** Дальний пир чужой пузырь **пропускает** (считает по тем комнатам грубое, что и так обязан).
8 игроков врозь = ~1 пузырь на машину, не 8 → снимается N-множитель с доминирующей (привязанной к слабейшему пиру) цены.

**УСЛОВНО** — держится только при одной смене смысла + только для геймплея, реально живущего на уровне комнаты (§4).

## 2. Модель авторитета — два слоя, граница на уровне КОМНАТЫ

1. **ГРУБЫЙ слой = ЕДИНСТВЕННАЯ общая правда.** Считается одинаково на каждом пире, каждый тик, для каждой комнаты (цена ∝
   активным комнатам, не вокселям). **Только он покрыт checksum** (масса по типам/слоям, температура {energy,capacity},
   cut-set-поток на рёбрах, region_id, resolution_tag, awake-set в каноне, доза-лута D6). Десинк определён ТОЛЬКО здесь.
2. **ДЕТАЛЬНЫЙ слой (пузырь) = ЛОКАЛЬНОЕ уточнение,** только где свой игрок. **НЕ в checksum,** легитимно существует на одном
   пире и нет на другом. Его единственная роль — богатое ощущение вблизи; в общую правду НЕ пишет.
3. **СОБЫТИЯ = мост.** Любое под-комнатное с ЖЁСТКИМ общим последствием (реакция бьёт всех; газ протолкнут в соседнюю комнату;
   взрыв) **промоутится в грубое авторитетное СОБЫТИЕ** — считается из грубого состояния, одинаково, на каждом пире, в checksum.
   (Это ровно интуиция владельца «глобальная симуляция + рассылает события».)

**Два жёстких правила безопасности:** (a) деталь течёт В ОДНУ сторону — читает грубое, никогда не пишет назад (ушёл из
комнаты → пузырь ВЫБРАСЫВАЕТСЯ, не collapse'ится в авторитет); (b) НОЛЬ нового трафика — про деталь/события по сети ничего не
едет; провод = только инпуты (D1). Дешёвая модель совместима с D1 ⟺ грубое самодостаточно для каждого общего последствия.

## 3. Условие «сглаживания» (когда дальний пир безопасно пропускает чужой пузырь)

**Точно:** прогнать деталь по комнате и свернуть к тоталам комнаты = ТЕ ЖЕ грубые числа, бит-в-бит, каждый тик, что и просто
грубая симуляция этой комнаты (включая вход/выход игрока, пробой стены). Просто: фокусы детали (импульс, вихрь, раздвоение)
могут двигать газ ВНУТРИ комнаты, но при сложении на уровне комнаты обязаны дать то же, что простая комнатная модель.
**Покупается инсайтом владельца:** раздвоения соединяются, импульс сбрасывается на границе → на уровне комнаты сглаживается.

**ЖЁСТКИЙ ОТКАЗ (честно):** если деталь может протолкнуть газ через дверь БЫСТРЕЕ/иначе, чем простая комнатная модель
(направленная струя, фронт взрыва, «продуть канал») → сосед получает другой приток → дальний пир (грубо) и ближний (деталь)
РАСХОДЯТСЯ. Два легальных выхода, третьего нет: (1) **зажать** меж-комнатный перенос детали к грубому закону (импульс =
чисто внутри-комнатное «ощущение», весь поток между комнатами — через тот же целый грубый edge-flux у всех); или (2)
**промоутить** в грубое событие (целый wind/edge-модификатор на ребре, выводимый из ГРУБОГО состояния, не из приватного импульса).

## 4. ⭐ БОЛЬШАЯ РАЗВИЛКА (фундаментальнее сети) — решает ВЛАДЕЛЕЦ

Каждый шаг, делающий модель дешёвой+безопасной, толкает общие решения ВНИЗ в грубый слой → и толкает детальный 3D-газ к роли
«только ощущение». Применённая бездумно, дешёвая модель ТИХО решает, что глубина геймплея = на уровне комнаты, а под-комнатная
форма облака = красивость. **Это решение — твоё, явно:**

- **ВАРИАНТ A — глубина НА УРОВНЕ КОМНАТЫ:** сколько какого газа в комнате, реагирует ли, бьёт ли стоящих, течёт ли в соседнюю,
  можно ли провентилировать/продуть комната-в-комнату. 3D-деталь = шикарное ощущение СВЕРХУ. → Дешёвая модель ОТЛИЧНА **и сильно
  ДЕ-РИСКует движок**: деталь может быть даже **float/по-пирно-разной/недетерминированной** (она выброшена, не авторитетна) →
  страшная проба «re-flux бит-в-бит по движущейся границе» (сам грид/граф-док звал её «клейм, не пруф, труднее под грид-first»)
  **падает до owner-eye проверки ощущения.** SS13 в основном комнатно-гранулярна — и глубока. Это НЕ мелкая игра.
- **ВАРИАНТ B — глубина в ПОД-КОМНАТНОЙ ФОРМЕ:** точная геометрия облака решает исход (проскочить сквозь тонкое место; продуть
  точный канал; стойкое раздвоение, меняющее, в какую дверь уйдёт газ и кого накроет). → Для ЭТИХ механик дешёвая модель
  НЕБЕЗОПАСНА: их последствие не вывести из грубого → форсит «каждый пир считает деталь этой комнаты» (дорого, по слабейшему) или
  десинк. Иметь можно — но только в комнатах, где платят все, и приняв предел.

**Практический путь — классифицировать КАЖДУЮ названную механику в 3 ведра** (ДО подписи ADR-0010):
1. **сглаживается → бесплатное локальное ощущение** (вихрь; раздвоение, что сливается в комнате);
2. **грубо-промоутируемо → дешёвое общее событие** (реакция; вентиляция комната-в-комнату; урон-где-газ);
3. **реально под-комнатное И общее → дорогой случай** (стойкое раздвоение, меняющее дверь выхода; продув точного канала).

**Первый проход по твоим примерам (реагируй/правь):**
- газ наполняет комнату / где сколько → **(coarse, ведро-1/общее)** ✓ дёшево
- реакция (бах): триггер+масс-баланс **(событие, ведро-2)**; телеграф-мигание/линия смешения **(ощущение, ведро-1)**
- урон от резидентного газа **(событие, ведро-2)**
- высота / низкая дверь сливает низ: влияет на приток соседа → грубый слой должен нести **несколько верт. слоёв** для
  правильной маршрутизации по высоте **(ведро-2; грубой высоты, не полного 3D)**; полное 3D-расслоение = ощущение
- прошёл сквозь → раздвоилось: если сливается **(ведро-1)**; если стойко меняет дверь/кого накрыло **(ведро-3, дорого)**
- продул воздухом / канал: вентиляция комната-в-комнату **(ведро-2)**; точный под-комнатный канал, решающий исход **(ведро-3)**
- выдавливание объектом: внутри комнаты **(ведро-1)**; протолкнул через дверь → через грубый edge-flux **(ведро-2)**

Вывод: похоже, игра **в основном A (комнатно-гранулярна) + пара под-комнатных механик** → дешёвая модель работает отлично,
классифицируем немногие ведро-3. Но это **твоё** подтверждение.

## 5. Сверка с грид/граф (честно — это РЕ-КЛАССИФИКАЦИЯ, нужна подпись)

Грид/граф-резолюция: ближний грид **авторитетен**, грубое = roll-up С НЕГО (fine→coarse). Буквально это значит «грубая правда
ЗАВИСИТ от того, что кто-то считает деталь» → форсило бы все пузыри везде. **Разворот (один флип):** грубое авторитетно везде и
считается самостоятельно; деталь — производное ОТ грубого, выбрасываемое на выходе; в общую правду не пишет, лишь обязана
СОВПАДАТЬ с ней там, где есть. Направление данных переворачивается: **«fine кормит coarse» → «coarse — правда, fine — выводимый
вид».**
Следствия: (1) `collapse()` из обязательного конвейера → **проверка согласованности** (или выброс на выходе); на дальнем пире
не происходит. (2) **Связующая проба МЕНЯЕТ работу:** binding-детерминизм = «грубое бит-идентично на 2 машинах» (тривиально) +
«expand(coarse) воспроизводим / co-located пузыри совпадают», **а НЕ re-flux-равенство**. Re-flux сохраняется, но понижен до
owner-eye no-pop ощущения — **КРОМЕ** D5-пространственных комнат и любых ведро-3 механик, где снова становится жёстким гейтом.
(3) D5: грубое значение комнаты авторитетно независимо от уточнения; fine-тень (если хранить) — только для формы на возврате.
По ZERO-LEGACY: чей тир теряет работу — его остатки удалить начисто.

## 6. Честные оговорки (несущие)

- **D5 большие комнаты (ангар) частично съедают выигрыш:** их много-клеточное состояние авторитетно → считается на КАЖДОМ пире,
  кто бы там ни был. 8× относится к мелким/схлопываемым комнатам; при нескольких больших «все-платят-пространственно» может
  доминировать. Ангар — отдельная Wave-A проба (масштаб клетки + деградация), ортогонально.
- **Co-located игроки видят РАЗНЫЙ газ молча**, если пузырь — не ЧИСТАЯ детерминированная функция (grub-состояние+region+tick+seed,
  целое, без локальной истории/float/порядка-контейнера). «Свободно-живущий пушистый пузырь» — ЗАПРЕЩЁН (если деталь обязана быть
  согласованной). *(При варианте A с чисто-презентационной деталью расхождение co-located = косметика, допустимо.)* Отладочный
  detail-hash для ловли в тестах.
- **Тихий десинк:** деталь намеренно вне checksum → баг collapse(detail)≠coarse-direct не роняет стоп, всплывает позже.
  Обязательно: ближний пир каждый тик локально проверяет collapse(detail)==coarse-direct и в checksum кормит только согласованное.
- **Анти-чит слабее** D1: деталь вне «судьи» → модифицированный клиент видит газ, которого не должен (gas-ESP). ОК для коопа с
  друзьями; записать, не утверждать «свойства доверия D1 сохранены».
- **Промо-события воюют с экономией сна:** непрерывная манипуляция держит комнаты «бодрыми» на всех → бюджетировать активную
  манипуляцию; события обязаны слушаться dirty-set.
- **Прецедента точной связки нет** → зарабатывается спайком; Wave A обязана уметь откатиться к «все-пиры-все-пузыри».

- **Грей-бокс (t-1): + ОБЩАЯ ось** — гонять как 2 инстанса, где игрок B переживает последствие манипуляции игрока A; помечать
  каждое поведение SHARED vs LOCAL. «Весело только манипулятору» = жёлтый флаг коопа.
- **Связующие пробы (t-2):** binding-детерминизм = «грубое бит-идентично» + «co-located пузыри воспроизводимы», НЕ re-flux.
  Re-flux → no-pop ощущение (кроме D5/ведро-3 → жёсткий гейт). + локальный tripwire collapse(detail)==coarse-direct. Ангар-проба
  с активной манипуляцией.
- **§9 швы:** в основном целы. §9.4 collapse → «выброс на выходе», не авторитетная конверсия. §9.5 checksum: coarse+region_id+

- **Визуал (g-7e15/RN1):** пер-клетка read-model — ТОЛЬКО для комнат, что уточняет локальный игрок; прочие — грубо. Удалённый
  детальный вид (камера/спектатор) = локальный expand(coarse) по запросу (view-only).
- **D5 возврат:** новое feel-решение — помнит ли отложенная комната ФОРМУ облака (sparse fine-тень, память на слабейшем) или

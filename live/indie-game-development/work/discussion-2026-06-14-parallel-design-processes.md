## ДИЗАЙН НА СЕЙЧАС (сжато, но полно)

### A. Процесс мира/лора — «мир-первым», сверху вниз (v3)
- Канон строится как **дерево сверху вниз** с правилом «ребёнок не замораживается раньше родителя».
- Порядок ствола: **L0 премиса/тон → L1 происхождение мира → L2 место (подземный город) → L3 кто мы (экипаж) → L4 силы/ставки → L5 вещество и системы (ВЕТКА, не ствол) → L6 листья по требованию.** Газ — следствие мира, не стартовая посылка; может даже называться не «газ».
- **Гейт N (для лора):** непротиворечиво (структурно) · не «за уши» (закон Сандерсона: необъяснённое не решает конфликт) · загадочно (у раскрываемой загадки заранее есть реальное «дно»; айсберг; вечно-открытые загадки помечаются).
- Канон-записи оформлены так, что **сами служат контекстом для AI** (инварианты вверху, JIT-подгрузка только нужных родителей) — чтобы ChatGPT и Claude не противоречили друг другу.

### B. Оценка механик — второй ствол + «шов»
- Рядом с лор-стволом растёт **ствол механик** с корнем **EXPERIENCE PILLARS** (3–5 целевых ощущений, эталон для механик).
- **Вещество — это ШОВ:** один узел оценивается дважды (как канон — гейт N; как механика — гейт M), наследует И лор-факт, И пилон; замораживается только если обе печати зелёные.
- **Гейт M (для механик):** интересное решение (Мейер: нет доминирующей опции, есть размен, ситуативно) · глубина через эмерджентность, не сложность · нет анти-паттернов (доминантная стратегия/вырожденная линия) · крючок (страх/жадность/мастерство/кооп + открытие/самовыражение) · кооп-взаимозависимость (нельзя пройти соло) · соответствие лору и пилону.
- **Честная граница:** бумага и agent-self-play ловят анти-паттерны и дают «можно прототипировать», но **«весело» сертифицирует только Steam Playtest.**
- **Path of Exile 2** — только как *настроение/принципы* для пилона ЖАДНОСТЬ (экономика игроков, высокая вариативность, валюта-как-крафт, комбинаторная экспрессия), НЕ копировать контент.

### C. Воронка генерации одного узла (вклад твоих идей O1–O7)
Порядок: **семена (разные премисы, в т.ч. абсурдная) → веер (1 агент/семя, кросс-семейно) → авто-триаж (дёшево выкинуть неспасаемое) → фильтр+ты (ночь инкубации, гейты, ОДИН вердикт) → слияние/доработка (скрестить карточки; только если добавляет решение) → решить+записать отказ → заморозить → рескан 1 шаг.**
- Разнообразие даёт **разность семян**, а не число агентов.
- **Журнал отказов** живёт хвостом `rejected_alternatives` в канон-записи (+ сырьё в history) — чтобы не предлагать мёртвое снова. Без 7-го типа файла.

### D. БОЛЬШАЯ ПЕРЕОЦЕНКА: слой `converge` уже есть в ядре
В OS появились базовые пьесы **converge / converge-arch / converge-verify** (+ `os/docs/converge-design.md`). Они уже дают структурный скелет, который я строил вручную: триаж по риску (=тиринг), рекурсивную декомпозицию (=фрактальное разворачивание), отдельную сессию-опровержение с НЕЗАВИСИМЫМ оракулом (=мой отдельный вердикт), канон→`knowledge/` + сборка→`work/converge-<node>.md`, батч ≤3 решений G7, retrofit/импорт, inline-отказы.
**Вывод:** не строить отдельную «design lane». Вместо — **гонять лор/дизайн/механики ЧЕРЕЗ converge** и добавить только то, что converge оставляет общим:
1. **Decision-class checklists = мои гейты N/M** (+ таксономия крючков, правила пилонов) — как записи `knowledge/`, которые читает converge-verify (он же их и авторит с нуля на первом прогоне; `knowledge/` сейчас пуст).
2. **Креативный front-end для RESOLVE** (разные семена, веер, слияние) — для строк «изобрести-а-не-вывести».
3. **Канон-как-payload** — форматная надстройка.
Плюс одна правка: для no-build лор-узлов терминус converge = заморозить канон в `knowledge/`, без shape/executor.
Также `work.md` получил шаг **«Owner inputs»** — он теперь кодирует со-творчество (owner-контент: сперва спросить тебя, строить из ответов, не выдумывать; недоступен → блок). Лор — это owner-контент.

### E. Использование платной ёмкости — «Флот подготовки решений» (последнее обсуждение)
- **Ключ:** узкое место — не compute, а **твоя пропускная способность решений + гейты.** Поэтому ёмкость покупает не параллельное исполнение, а **глубокий, опровергнутый, готовый-к-решению инвентарь.**
- **Модель:** много агентов дёшево готовят брифы (опции+цитаты+кросс-семейное опровержение) → гейты → **ПОЛКА ≤3** → ты сливаешь **≤3 решений/нед** тремя глаголами **SIGN** (в `knowledge/`) / **BANK** (re-park 1 строкой) / **DROP**. **Обратное давление:** полка полна → флот СТОП. Лимитируем слив, не машины.
- **Правило что открывать:** только линии с близким реальным сроком, который «съест» результат. Плоский веер = документы; календарный = прогресс.
- **Открыть в этом месяце:** g-5b07 (Steam/Fest/деньги — даты 31авг/5окт/11дек), брифы по ассетам/тех-покупкам (vs кэп $3k), авторинг decision-class checklists, тонкая *рамка вопросов* геймдизайна.
- **Держать:** g-7e15 арт (пока не залочен render-seam), g-e6f2 маркетинг (E2 не приоритет + веха «первый клип» снята), лор-контент (со-творчество).
- **«Команда»** = вызываемые роли-шаблоны под линзы (curator/market-scout/asset-scout/refuter/checklist-author/librarian), не постоянная студия; номинальная группа, кросс-семейная проверка. Включать 2–4, не 7.
- **OS-чистота:** informs-never-writes; никогда не второй активный bet; канон только на SIGN с real read_by; брифы = `work/decision-prep-<lane>.md`; постоянного файла-очереди НЕТ; kill линию после 2 пустых сливов; правило «≤10% часов» перечитывается как лимит *слива* → **нужна твоя явная ратификация** + завязано на map-re-check (веха «первый клип» снята).

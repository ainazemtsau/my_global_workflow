## 0. Что это
Выход обсуждения газ-модели. НЕ ТЗ — конструкция, к которой пришли владелец + Claude.
Проверено двумя состязательными аудитами и глубоким мат-ресёрчем. Дилбрейкеров нет.
Возврат: ревью d-gasmodel-redesign-001. Планировать по этому документу по кускам.

## 1. Вердикт одной строкой
Недовольство было про МОДЕЛЬ (что газ умеет / где он), не про картинку. Решение —
дорога A+ (оставить граф-основу, нарастить слои), НЕ B (переписать в непрерывное поле):
A+ честно даёт кооп-детерминизм, B заново открывает дорогую сетевую проблему.
Это УКРЕПЛЯЕТ уже ратифицированную архитектуру репо, а не сносит её.

## 2. Полная конструкция (целевое решение)

### 2.1 Мир и резка
- Мир = осевые прямоугольные «коробки» по сетке 25/50 см. Шахты, коридоры, ангары,
  этажи, ямы, помосты — всё коробки.
- Сложные комнаты (Г-образные, с двумя входами, ангары) режутся на выпуклые куски
  ПРИ ГЕНЕРАЦИИ (детерминированные данные). Куски одной комнаты ПОЛНОСТЬЮ открыты
  друг другу (резка = разрешение, не стены). Разные комнаты связаны только проёмами.

### 2.2 Основа = «вода»: сохраняющийся газ, текущий по графу
- Комната/кусок = узел с ЦЕЛЫМ количеством газа по 2-3 слоям высоты, на каждый тип газа.
- Проём = ребро с ВЫСОТОЙ (sill): низкая дверь пускает тяжёлый, высокое окно — лёгкий,
  вертикаль (шахта/дыра) = тяжёлый вниз + лёгкий вверх встречно; бит «открыт/закрыт»;
  проводимость (ширина).
- Правило за тик (~1 Гц/по событию): через каждое открытое ребро перелить ОГРАНИЧЕННУЮ
  долю из узла-где-выше в узел-где-ниже. Газ только ПЕРЕМЕЩАЕТСЯ → сохранение точное,
  бит-в-бит. Останавливается, когда уровни сравнялись.
- Из ОДНОГО правила само: налив, перелив-через-высоту, «нашёл уровень», слив/вентиляция/
  пролом/ОТСТУПЛЕНИЕ/выдохся-источник. Никакого спецкода на «слив».
- Яма = ловушка тяжёлого (копится до края, переливается). Помост = сухая высокая земля.
- Источник = кран (конечный/бесконечный — оба).
- СОН: узел спит при нуле работы, КОГДА в нём нет живого взаимодействия. (см. 2.6)

### 2.3 Плавучесть и вентиляция
- Каждый газ ищет СВОЙ уровень по весу: тяжелее воздуха → пол, легче → потолок,
  нейтральный → середина. Слои (пол/середина/потолок) = это «вверх-вниз».
- Слой не прибит навсегда: вес + ТЕМПЕРАТУРА (нагрел тяжёлый → всплыл, остыл → осел).
  Температура — отдельный тонкий слой.
- Вентиляция (позже) = «труба с насосом»: принудительный поток на ребре поверх
  естественного; может переспорить плавучесть. Встаёт на тот же граф без переделки.

### 2.4 Фронт = где сейчас передний край
- Целочисленное «геодезическое расстояние» от источника (Дейкстра по графу проёмов,
  с учётом стен и высоты). «Фронт дошёл до точки» = одно сравнение. Сквозь стену не врёт
  (ребра нет). Тонкий слой ПОВЕРХ массы (масса = правда, фронт = край).
- Внутрикомнатный 1-мерный «докуда дополз» — НЕ-авторитетная подсказка.

### 2.5 Зерно и сон — по ИНТЕРЕСУ (не только по игроку)
- Холодный узел (нет интереса/игрока) = только грубая «вода».
- Тонкое локальное зерно (~1 м у интереса, 25 см у игрока) поднимается только там и
  кормится РЕАЛЬНЫМ фронтом (не размазкой средней по комнате).
- Узел НЕ СПИТ, пока идёт живое взаимодействие (газ реально что-то обрабатывает здесь).
  Это убирает «досчёт дозы во сне» как класс: поломка и её каскад — обычные живые события.
- Дисциплина: «не спящих» и «интерес» держать РЕДКИМИ, иначе экономия рушится.

### 2.6 Лут / точки интереса — железный закон
- Урон = от газа, РЕАЛЬНО СТОЯЩЕГО на предмете, по времени. Нет пересечения → 0 урона →
  то, чего газ не коснулся, не ломается НИКОГДА.
- Числа (скорость дозы, порог, затухание) — КОНФИГ. Источник дозы (резидентный газ, не
  «фронт лизнул») — ПРАВИЛО, не крутится.
- Вблизи (игрок) — по клеткам, точно. Вдали — по «куску» (~1 м), погрешность допустима.
- Поломка — истинное прошлое событие; газ мог потом отступить/развеяться — это правда,
  не дорисовываем. Урон и картинка читают ОДИН газ → «сломалось, а газа нет» невозможно.
- Тот же крючок: скользкий пол, прочие интерактивные предметы.

### 2.7 Встреча двух газов
- Эмерджентно: оба пришли в один кусок → событие встречи в тик второго прихода.
- Точная координата (если рядом игрок / реальная реакция) → этот один кусок в клетки,
  ищем линию контакта.
- Необратимый взрыв жмём ТОЛЬКО после подтверждения со-расположения на достаточном
  зерне → грубое в одиночку ложный взрыв не выдаёт.

### 2.8 Игрок в поле (закрыло главную дыру аудита)
- Игрок ВСЕГДА на детальном уровне. Урон И ВИДИМОСТЬ для него — по клеткам = по тому,
  что он видит. «Умер в чистом воздухе» / «попал сквозь дым-укрытие» невозможны.
- Не два source-of-truth: ОДНА правда на двух приближениях (детальное = грубое, зум,
  в сумме то же, не противоречит).
- Кооп: детальное считает ХОСТ одним общим полем (клетка детальна, если рядом ЛЮБОЙ
  игрок) → двое в одной точке видят одно и то же → видимость безопасна как ПРАВДА.

### 2.9 Взрывы / цепь / огонь
- Взрыв (первая механика) = мгновенный «бум» в точке встречи/поджога.
- Цепь: вторичные взрывы — СЛЕДУЮЩИМ тиком (ограничено, детерминированно, читается как
  ползущий фронт). Без рекурсии в одном тике (иначе детонация целого крыла за тик = фриз).
- «Поджог» — бегущий огонь по газовому следу — ПОЗЖЕ, отдельная механика на том же
  «фронте». Сейчас не нужен.

### 2.10 Разрушаемость (сложная — БЕЗ переделки)
- Дырки любого размера, проломы, обвал-открывающий-проход, выбить стену целиком, рост
  дырки = добавить/расширить/щёлкнуть РЕБРО. Выбил стену = широкий проём → как одна комната.
- Новая ПРЕГРАДА (завал делит комнату) = режем линии разлома ЗАРАНЕЕ при генерации,
  в рантайме щёлкаем внутренние связи. Генератор разрушаемое и так знает.
- ИНВАРИАНТ: без переразбивки геометрии в рантайме (убирает класс багов деления массы).

### 2.11 Детерминизм / сеть
- Lockstep, ОДИН хост — авторитетный писатель, без сервера; клиенты РЕКОНСТРУИРУЮТ,
  не пересчитывают.
- ВСЁ авторитетное/реплицируемое — целые/фикс-точка, бит-в-бит. Float только как
  внутренний расчёт, детерминированно сведён к целому до хеша/провода.
- ДЕЛАЙ ТАК: целочисленная графовая математика (целый Дейкстра + целый поток).
  НЕ ТАК: учебный плавающий решатель давления (Newton-Raphson), непрерывный Eikonal с
  корнями — молча ломают бит-в-бит. (Возьмёт инженер готовый солвер вентиляции —
  детерминизм умрёт на первом тике.)
- Геометрия/резка — данные с генерации. Синк маленький (события + целые по узлам).

### 2.12 Мультиплеер 4-8
- Хост = грубая «вода» по всему уровню + малый детальный «пузырь» у каждого КЛАСТЕРА
  игроков (стоящие рядом дедуплицируются в один). 8 разбежавшихся = ЛУЧШИЙ случай
  (8 дешёвых холодных пузырей).
- Реальный драйвер цены = пузыри-на-АКТИВНЫХ-событиях (8 игроков у 8 разных живых
  фронтов), не разбежка. Нужен потолок суммарных детальных клеток-за-тик + честный
  круг-по-очереди; политика деградации аплинка (события — приоритет/надёжно; косметика —
  коалесинг + по интересу).

## 3. Принцип «конфиг vs правило»
- В КОНФИГ: все числа (дозы, пороги, затухание, число слоёв на комнату, проводимости).
- ФИКСИРОВАНО (правила корректности): доза от резидентного газа; никакого float в
  авторитетном состоянии; реакция жмётся только на подтверждённом со-расположении.

## 4. Чем подтверждено
- Аудит №1 (архитектура): держится, дилбрейкеров нет; совпало с ратифицированной
  моделью репо (многозонный граф + полосы + sillZ-проёмы + реконструкция).
- Мат-ресёрч: «пространство как данные» = сохраняющийся граф + геодезический фронт —
  лучший фит; поправка — НЕ односторонний растущий фронт, а конечное сохраняющееся
  количество (вода). Детерминизм — целым графом, не float-солвером.
- Аудит №2 (геймплей): все 6 направлений держатся, дилбрейкеров нет. «Вода» = корона.

## 5. ОБЯЗАТЕЛЬНО проверить нагрузкой ДО ставки
- Каскад: один поджог в длинное газонаполненное крыло (~60 кусков) — тик ограничен,
  глубина идентична на 2 машинах (правило «след. тиком» + бюджет событий).
- 8 горячих пузырей: 8 игроков у разных активных фронтов — кадр в потолок, круг-по-очереди
  держит правду.
- Фронт в сложной комнате на экране — огибает угол, не «прямая стена», без излома на швах.
- Целочисленный инкрементальный Дейкстра на проломе — ограничен и детерминирован
  (полудневный проб).
- Сохранение массы ПРЯМО на тике пролома + взрыв-как-перенос (RED-тест: любая операция,
  меняющая суммарную массу, = баг, не баланс).
- Высокий зал на экране ДО выбора числа слоёв (глазами; слои = данные на комнату).
- Нет «попа»/мерцания на перерождении грубое↔детальное (рендер-кроссфейд).

## 6. Открытые куски (ещё НЕ разобрали — для планирования)
- Операционка: сохранение/перезагрузка состояния; ПОТЕРЯ ХОСТА (миграция авторитета —
  иначе падение хоста = смерть сессии 4-8).
- Движущаяся геометрия: лифты, бласт-дверь, что закрывается в потоке и режет газовый
  столб (сохранение массы в момент движения).
- Грифинг: детерминированная симуляция в руках игроков = набор примитивов против
  напарника (загазовать, перекрыть вентиляцию, заманить-в-яму-и-поджечь). Не осознано.
- Экономия грубого слоя при МНОГИХ мелких источниках / повсеместной утечке (чтоб «спящие
  = ноль» не рухнуло на always-on слое).
- Believability/READABILITY как РЕНДЕР-задача (слои→туман без «полок», читаемость какой
  газ/где опасно/где фронт). Симуляция данные отдаёт; это художественный проход.

## 7. Решения владельца на потом (вкус/крафт)
- Подпись каждого газа (цвет/движение/край) — отдельная большая художка, не сейчас.
- Число слоёв: 3 по умолчанию, 4-5 в высоких витринных залах (подобрать глазами).
- Тайминг огня — только КОГДА появится поджог (мгновенно vs ползёт).
- Насколько тянуть «правый угол» реконструкции.

## 8. Развилка d-gasmodel-redesign-001 — куда склонилось (решение — за владельцем)
- РЕКОМЕНДАЦИЯ разбора: дорога A+ (граф-основа + слои), не B (непрерывное поле).
- «Убрать ёмкость+перелив» переосмыслено: это видимое лицо настоящего давления (оно уже
  есть) + намеренное «угроза не стравливается сама»; полностью убирать без закона-замены
  нельзя (иначе безграничный/осциллирующий газ → перф рушится). Реальная боль (потерял
  «где») лечится фронтом+зерном, а не удалением ёмкости.
- «Свободное число газов» бьёт по доказанной сети → нужна дисциплина «ограниченный список
  активных газов на узел + жёсткий потолок на проводе» («как богато ОЩУЩАЕТСЯ» ≠ «что летит
  по сети»).
- Проб C (1-2 нед сетевой) нужен ТОЛЬКО если всё ещё хотим непрерывное поле; разбор
  показывает, что не нужно.

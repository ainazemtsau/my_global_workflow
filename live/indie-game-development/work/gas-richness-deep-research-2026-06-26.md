**Дата:** 2026-06-26. **Триггер:** owner оспорил вывод «разделение нельзя» и попросил deep research по всем подходам (включая экзотику) + собрать инновационное/гибридное решение; явно назвал газ «сердцем», отказался от жёстких компромиссов. **Метод:** workflow `gas-heart-deep-research` (20 агентов, ~2M токенов, веб-поиск + чтение кода): 7 углов ландшафта → 3 архитектора собрали кандидатов → скептики разнесли каждого по детерминизму/перфу/«даёт ли реально» → судья свёл. Источник истины — этот файл; полный сырой вывод был в task wf7a7tcjb.

## Ландшафт (что вообще существует)

- **Lattice-Gas CA (FHP/HPP) и Integer Lattice-Boltzmann (ILG):** единственное полностью ЦЕЛОЧИСЛЕННОЕ семейство, где импульс/направление — это хранимое состояние → даёт ветер, адвекцию, СТОЙКОЕ разделение и НАСТОЯЩИЕ вихри как общую правду. Детерминизм ОК (битовая логика), но FHP-столкновения требуют seeded-PRNG в канонном порядке; есть «шумовой налог» (нужно усреднение); 3D-изотропность требует FCHC (24 бита/клетку — дорого). Вывод: идеал «богатого целочисленного 3D», но как ГЛОБАЛЬНЫЙ солвер на слабом железе не тянет → годится ЛОКАЛЬНО (в спец-комнатах).
- **Grid-velocity + fixed-point (Stable Fluids / NS в Q-формате):** настоящее поле скоростей в fixed-point целых → настоящая инерция, ветер, разделение. Детерминизм достижим (Photon Quantum, StarCraft, FixPointCS — прецеденты), но «по аудиту, не по построению» (деление с зависящим от данных округлением). Дорого по памяти на уже memory-bound ядре. = «потолок», не ближний билд.
- **Particle (SPH/PBF/FLIP/MPM):** богато, но float-натив, детерминизм тяжёлый, дорого → для авторитетного пути не подходит (только косметика).
- **Hybrid / event-impulse (ИДЕЯ OWNER):** диффузия-база + детерминированные целочисленные СОБЫТИЯ-импульсы (направленный «кик» по грани/линии) + анизотропная проводимость / односторонние клапаны. Самый дешёвый инновационный путь; даёт ветер и направленность БЕЗ векторного поля.
- **Библиотеки:** готового fixed-point ФЛЮИД-солвера НЕ существует (любой векторный тир — bespoke). FixPointCS (MIT), FixedMathSharp (MIT) — только арифметика, если когда-нибудь делаем вектора. Unity.Burst/Jobs (уже в проекте) — рычаг для отложенного sparse-солвера. Float-флюид плагины (Zibra/Obi) — ТОЛЬКО косметический визуал-слой, НИКОГДА не авторитет.

## Три кандидата (собраны архитекторами, проверены скептиками)

1. **Forced-Flow Hybrid** — затухающий знаковый ЦЕЛОЧИСЛЕННЫЙ регистр направленного смещения (bias) на грань + импульс-события. **РЕКОМЕНДОВАН.** Внутри замкнутой архитектуры, без вскрытия замка, без float, без нового RNG, дешевле всех. По сути — это и есть S1 (c-exec-015), обобщённый в «слой источников импульса». Даёт ветер авторитетно + затухающий «толчок» (hint инерции). Честен, что вечное разделение/вихри отложены.
2. **Momentum Tokens + Detail-Bubble Lattice (MT+DBL)** — близнец №1 в ближнем тире, НО добавляет явный тир эскалации: seeded-целочисленный lattice-gas в НЕСКОЛЬКИХ спец-комнатах (bucket-3) для настоящих эмерджентных ВИХРЕЙ как общей правды, без float. Лучшая «дорожная карта» дальних случаев. Тир 2 не построен, ADR-класс, ждёт sparse-солвера.
3. **Drift Lattice (fixed-point вектора)** — настоящая хранимая инерция + плавно затухающее стойкое разделение как общая правда. Но bespoke, без готовых решений, тяжелее всех по памяти, и — главное — ТРЕБУЕТ owner-подписанного вскрытия замка ADR-0010 (кандидат это ошибочно отрицал; роадмап §52 и SPEC §3 называют velocity/advection STOP-эскалацией). «Потолок», не ближний билд.

## Рекомендация (бленд)

- **СТРОИМ СЕЙЧАС (= S1, без вскрытия замка):** один затухающий знаковый int-регистр на ОТКРЫТУЮ грань рядом с flux/carry; пишется импульс-СОБЫТИЯМИ (спавн-выброс, толчок телом, вентиляция, граната — всё едет по существующему lockstep-input-каналу, нулевой новый трафик); применяется как один лишний clamped целочисленный перенос поверх градиента; затухает к нулю. Даёт авторитетный ВЕТЕР + затухающий толчок дёшево и доказуемо-безопасно.
- **РАМКА (из MT+DBL):** называем эскалацию явно. Настоящие вихри как общая правда = seeded-int lattice-gas в горстке bucket-3 комнат, после sparse-солвера, по ADR. Везде ещё — вихри в косметический g-7e15 слой, накормленный bias-полем (свирл выровнен по реальному ветру, нулевой риск).
- **РЕЗЕРВ (из Drift Lattice):** fixed-point векторное поле — именованный апгрейд-тир, триггер ТОЛЬКО если owner-eye скажет «затухающий bias недостаточно живой», через owner-подписанный ADR-0010.

## 4 нагруженные поправки (вложить в PLAN до первой строки кода)

1. **Trip-wire §9.5:** суб-гранёвый bitmap живёт в StructureGrid (отдельный слой, свой чексумм, bitmap ВНЕ газового чексумма), НЕ в VoxelField. Первый инкремент bias держать ГРАНЁВО-РАВНОМЕРНЫМ (функция уже-чексуммленного COUNT проводимости + гранёвый DirectionMask-скаляр) → остаётся в газовом чексумме. Суб-гранёвую направленность (клапан на нижней половине двери) ОТЛОЖИТЬ в отдельный слайс через §9.7 committed-revision read-model.
2. **Арифметика затухания:** знаковый сдвиг (`bias -= sign*max(1,|bias|>>k)`) релаксит +bias и −bias асимметрично → молча ломает направленную симметрию. Пин симметричное целочисленное правило, доказать в RED-оракуле.
3. **Доказательство контракции ПРИОСТАНОВЛЕНО, не сохранено:** экзогенный bias — не функция градиента, в форс-окне клетка может превысить макс окрестности. Честно: «возвращается к контрактивной базе после затухания», гейтится RED non-monotone оракулом (ветер-в-угол-и-обратно ОБЯЗАН сохранить массу И осесть, НЕ осциллировать), который ОБЯЗАН падать на плохом bias. Взаимодействие clamp×decay (отклонённый клапом перенос дропается, пока регистр ещё затухает) проработать в оракуле, не утверждать.
4. **Раздувание active-set:** bias держит грань активной ~N тиков → ест экономию ОТЛОЖЕННОГО sparse-солвера в том самом режиме, где решается бюджет слабого ядра. Cost-assertion в оракул: кикнутый регион обязан вернуться в «тихий» набор за ограниченный бюджет тиков. Перемер ангара делать ПОД ФОРСИНГОМ (стоячая вентиляция), не по settled-пробе.

## Дешёвый прототип-спайк (де-риск до коммита)
(a) RED non-monotone settle-оракул + симметричное затухание + clamp×decay сохранение + bounded-return-to-quiet — снимает поправки 2/3/4 разом (аналог уже решённого Kp=4→12).
(b) 2-процессный loopback bit-identity хеш с bias в MeaningChecksum + ОДНОВРЕМЕННЫЕ импульс-записи в ОДНУ грань (две гранаты в тик) → пин канонного порядко-НЕЗАВИСИМОго правила записи (единственный открытый шов детерминизма).
(c) Owner-eye на форкнутом коридоре + вентиляция: «ветер несёт газ, резко разделяет, держит сплит пока форсим, с ощутимым толчком, что гаснет». Если читается живым — дорогие тиры могут не понадобиться.

## Честные пределы (не переобещать)
- Дешёвый тир авторитетно даёт ТОЛЬКО ветер + затухающий толчок. Развилка/стойкий-сплит — транзиентно (пока форсим), сливаются на затухании. Настоящая инерция/вечный-сплит = fixed-point (вскрытие замка). Свободные вихри = lattice-gas комнаты. Поставить owner-ожидание ДО билда.
- «Развилка-сплит» и «стойкий-сплит» — ОДНА физика, не два отдельных выигрыша. «Инерция» дёшево = затухающий порыв, не докатывание. (Поправка переобещаний кандидатов.)
- Большие залы по-прежнему гейтятся отложенным sparse-солвером (d-sparse-solver-defer-001); хвост затухания bias взаимодействует с ним → мерить под форсингом.

## Decision frame (what the research overturned)
The owner's first instinct + my first seed batch (real evocative words from the game's gas/industrial theme) mostly FAILED the availability/collision/longevity gates. Net finding: a real dictionary word — especially one tied to the game's theme — tends to be (a) already taken as a handle, (b) collision-prone, (c) weak for trademark, (d) theme-locked. The robust strategy is a SHORT, OBSCURE, OWNABLE word (coined or rare proper noun), theme-agnostic, name == handle. The owner steered to the Zarathustra/Zoroastrian well; **Chinvat** is the cleanest survivor of that well.

## Pass 1 — norms + 8 theme-seed collision check (wf_f125eacd, 12 agents; 1 lens died on 529)
**Naming norms (3 lenses, web-cited):**
- Suffix (Games/Studio/Interactive/Entertainment) is OPTIONAL; bare evocative word reads most confident; "Games"/"Studio" fit a solo "nobody"; AVOID "Interactive"/"Entertainment" (reads bigger/dated). A suffix on a common word adds SEO distinctiveness. (Game Developer "Diary of an Indie Studio"; ResetEra.)
- Short (≤~5 syllables), pronounceable; survive heard-out-loud + typed-from-memory. AVOID invented portmanteaus / cute misspellings (the "Plearn/Playcademy" trap).
- Generic/common-word names get buried in search, are hard to trademark (low distinctiveness), and collide → forced renames (real: inXile C&D over "Wasteland"; Steam-enforced "Hidden Cats" TM). Distinctive/arbitrary = #1 Google result + strongest mark.
- `.com` NOT essential in 2026 (.io/.games/.gg accepted); same-handle consistency > .com. Test handle/domain availability BEFORE committing (a hard gate experienced indies use).
- Trademark for micro-indie: free ™ by use; ® ~$275–350/class (or +$500–1500 attorney) — defer until commercial traction; do the FREE clearance (USPTO TESS + EUIPO + Google + Steam) before committing. Many acclaimed indies never register.
- Steam: Developer/Publisher are free-text, DISPLAY name editable later, but the LEGAL partner-entity name is sticky; the name is visible/clickable and powers the Creator Homepage (this is what makes ONE reusable studio name valuable — it aggregates the catalog, fans of game 1 find game 2). Anti-impersonation rule: your Creator URL is a "Company Mark" → clear collisions first.
- DON'T pigeonhole to a genre/theme — pick a name reusable across many future games.

**Handle reservability — intersection rule (binding, web-cited):** for ONE handle to stand on Steam vanity + X + Bluesky + Reddit + Discord + newsletter + itch.io + Ko-fi simultaneously it must be: coined/obscure, lowercase, pure ASCII letters (digits ok), **NO `_` and NO `-`** (X forbids `-`, Bluesky forbids `_` → no separator is universal), **4–15 chars** (X caps at 15; Substack floors at 4). Reddit username is PERMANENT (no rename). Checkers: namechk.com / instantusername.com / namecheckr — first pass only; MANUALLY confirm each critical handle (documented false positives). Studio name == handle is the low-friction default for a solo dev.

**Connotation (the strategic finding):** the studio name is NEAR-NEGLIGIBLE for the reach of game #1 — capsule + first-10-sec gameplay + tags do ~all the work (e.g. moving gameplay to second-1 = +31% wishlist conversion); players remember the GAME, not the studio; causality runs game→name. Brand consistency pays on game #2+ (12–20% return-visitor lift) → optimize for LONGEVITY + reservability + comfort, not "marketing punch." Ironic fake-corp name CAN be authentic ONLY IF (a) the joke is owned openly (Yacht Club test) AND (b) the public voice is the real first-person human (label = wink, not persona) — else it collapses into the corporate facade. KEEP the cold-corporate/ironic register for the IN-GAME world; keep the STUDIO name human/neutral. ("Acceptable Losses" = great in-game corp, poor studio name — and already a shipped game.)

Контроль-слой — как этим пользоваться

В любом новом чате работают фразы: «что можем делать» — покажет этот борд свежим; «делай рефакторинг» — запустит ночную линию по бэклогу; «дневное ревью» — сверка направления с записью находок; «давай пройдёмся по развилкам» — решения из «Ждёт тебя» по одному, с вариантами и рекомендациями; «обнови борд» — принудительная перегенерация панели.

Словарь борда

лег — одна порция инженерной работы, проходящая полный контроль качества (план → стройка → независимая проверка); примерно день. · dev / main — dev = единственный долгоживущий рабочий Unity+MCP workspace; main = чистая integration-площадка без прямой работы и Editor. · worktree — дополнительная рабочая копия одного Git-репозитория; имя папки не доказывает ветку или статус, поэтому authority хранится в product registry. · мерж-гейт — твоё «да» на вливание работы в основную ветку. · PLAN / BUILD — две отдельные сессии одной работы: сначала план (ты одобряешь и он замораживается), потом стройка строго по нему. · G5 / опровержение — независимая свежая проверка, которая ПЫТАЕТСЯ доказать, что сделано неправильно; не смогла — работа принята. · контрольная сумма (чексум) — число-«отпечаток» состояния симуляции; совпадает у всех игроков — значит игра синхронна. · ставка / аппетит / стоп-линия — текущая большая цель / её бюджет в легах / дата или лимит, после которого работа не продлевается молча. · спайк — быстрый бросовый эксперимент ради ответа на один вопрос: код выбрасывается, знание остаётся. · канон / рамка / якорь — закон игры / принятая основа дизайна (ещё не закон) / материал, который нравится и хранится, но не закон.

Ночной рефакторинг: процесс запуска

твоя команда: «делай рефакторинг» · мерж — утренним батчем

Вечером (ты, 1 минута)

  • Новый чат: «делай рефакторинг» — всё
  • Опционально: «только ядро» / «только доки» / «N штук»

Ночью (само)

  • Пред-полёт: читает борд/NOW — какие зоны СВОБОДНЫ от летящих легов; занятые зоны не трогает
  • Берёт верхние элементы из work/refactor/backlog.md, чьи зоны свободны
  • Каждый элемент — свой worktree/ветка, параллельно
  • Доказательство на каждом: голдены байт-в-байт + чексум + 1525 тестов + mutation не хуже + оба скана
  • Следом на каждый — G5-опровержение свежей сессией («поведение изменилось? найди») — тоже ночью
  • Что не доказалось — ОТКАТ и пометка в бэклоге, в отчёт честно

Утром (ты, ~5 минут)

  • Отчёт в борде: «доказано N, отклонено M, ждут мержа»
  • Мерж в main — твой батч (ворота не ослабляем): смотришь отчёт → «мержи» → очередь мержей по одному с полным прогоном

Границы ночного

  • Ночью — только headless-доказуемые зоны: ядро, тесты, tools, доки
  • Адаптер/рендер-рефакторы — дневные (нужен твой 5-мин глаз)
  • Строго ноль изменения поведения; никаких «попутных улучшений»
  • В main ночью не мержится НИЧЕГО
Для установки: (1) бэклог work/refactor/backlog.md — наполняется стартовым read-only сканом; (2) локальная пьеса plays/refactor-night.md с этим процессом; (3) строка-указатель в NOW, чтобы фраза «делай рефакторинг» из любого нового чата находила пьесу. Финишная сцена Полигона M1

главная цель · маршрут принят 14.07 · NearGasSimulationLab = внутренний gas-стенд
Одна сильная сцена вместо набора несвязанных песочниц: DA собирает минимальный проверенный module set; хотя бы один модуль проходит PGG editor-time bake; reusable player prefab подключён к авторитетной позе; газ имеет pause/manual-step и приборы; controlled breach и full flood наблюдаемы; один сценарий повторяется на двух машинах и слабом железе со snapshots/телеметрией.
Что остаётся дочерним: L3 NearGasSimulationLab строится внутри gas PROGRAM и доказывает газовую интеграцию. Он не считается автоматически всей M1-сценой. Принятый `work/poligon-m1-integration-route.md` фиксирует точные merge points, текущий 34+ ledger без cap и cuts; прежняя Phase 0 и старый level CALL не возвращаются.
Словарик: трек — отдельная текущая линия работы с собственными CALL; primary — трек текущей ставки, а default — CALL, который выберет «продолжаем»; WIP-лимит — сколько треков одновременно занимают рабочие слоты (ready, waiting, blocked и pending decision считаются, paused — нет); грейбокс — уровень из серых болванок без арта, чтобы проверить геймплей; standalone-билд — игра как отдельный .exe, а не запуск из редактора Unity; вертикальный срез — маленький кусок игры, где есть всё сразу (управление, цель, смерть, звук), а не одна система целиком; капсула — обложка игры в Steam, главный конверсионный арт; вишлист — «в желаемое» в Steam, главная валюта до релиза; слот Next Fest — право участвовать, даётся игре один раз навсегда; прототип-лента — отдельная «песочница» в проекте для быстрого кода оболочки, не трогающая проверенное ядро симуляции; телеметрия — автоматический сбор данных, как играют люди (сколько минут, где умерли); min-spec — слабый компьютер, на котором игра обязана работать.

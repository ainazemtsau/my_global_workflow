## 1. Что мы сейчас решаем

**g-9c41** — первый узел дерева направления, «хребет», на котором стоят все остальные. Его цель словами: должна существовать маленькая, но представительная сетевая кооперативная сцена, которая работает и не разваливается:

- 2–4 игрока, один из них — хост (его машина считает игру; отдельный сервер не нужен);
- симуляция нескольких газов на графе комнат с дверями и вентиляцией, авторитет у хоста;
- хотя бы одно «разрушение», которое меняет топологию (прорвало стену → газ пошёл по-новому);
- уровни подаются из процедурных генераторов через заменяемый адаптер;
- всё это поддерживаемо одним человеком.

**Что уже решено и в этой сессии НЕ пересматривается:**

- **Аппетит: 6 недель, жёсткий конец 2026-07-24.** Ты выбрал его сам в прошлой сессии (s-shape-001). Аппетит в этой системе не продлевается никогда (гейт G3): не успели — ставка умирает, продолжение возможно только через новый шейп новой ставки. Это защита от бесконечного «ещё чуть-чуть».
- **Внутренняя веха: первый зрелищный клип ~2026-07-10.** По решению карты (map_order) этот клип открывает параллельные треки: страницу Steam, каденцию постинга. До клипа эти треки получают ≤10% часов недели.
- **Репозиторий: GasCoopGame@236bc30e** — канонический продуктовый репозиторий во всех трёх подходах. Фундамент в нём (G4A) принят 2026-05-23: тестовый стенд, швы композиции, вход валидации, интерфейс топологии. Старый репозиторий TheLastExit — только источник свидетельств и референс: смотреть можно, код напрямую не переносится (запрет аудита A1).

**Что решается именно сейчас:** каким СПОСОБОМ добиваться цели узла за эти 6 недель. На столе три структурно разных подхода из s-shape-001 плюс мой заострённый гибрид. ## 2. Словарь — все термины простыми словами

| Термин | Что это |
|---|---|
| **room-graph / T1** | Симуляция на уровне «комната = один узел». У комнаты одна масса газа («в комнате 40 кг дыма»), комнаты соединены порталами (двери, венты), газ перетекает по порталам. T1 — этот уровень детализации. |
| **T2** | Внутрикомнатная детализация: газ внутри комнаты распределён по подъячейкам — видно, как облако ползёт по комнате от пролома к двери, а не просто «комната заполнена на 40%». Дороже в разработке и в сети. |
| **мета-газ / конфиг-газ** | Двухуровневая система типов газа из цели узла. Мета-газ = архетип поведения с собственным кодом-обработчиком (например «горючий», «оседающий»). Конкретный газ внутри архетипа = чистый конфиг (цифры, без кода). Проверка зрелости: третий газ добавляется ТОЛЬКО данными — если получилось, система правильная. |
| **chunked-delta поток** | Способ гнать газовое поле по сети: мир порезан на чанки, хост каждый тик шлёт только ИЗМЕНЕНИЯ изменившихся чанков, а не всё поле целиком. Так делают все работающие прецеденты (SS14, Stationeers, Noita-мод). Этот кастомный поток — и есть переиспользуемая технология, «moat». |
| **NfE** | Unity Netcode for Entities — сетевой стек Unity. По аудиту (поправка P6): вендор по умолчанию, но зафиксирован не вендор, а МОДЕЛЬ: симуляция авторитетна у хоста + кастомный chunked-delta поток для газа + сетевые «призраки» (ghosts) только для игроков и объектов. Вендора можно поменять (Netcode for GameObjects, FishNet), модель — нет. |
| **listen-server** | Один из игроков — хост, его машина и есть сервер. Выделенных серверов нет и не планируется (зафиксировано P6: 2–4 игрока, player-hosted, до самого EA). |
| **harness (стенд)** | Тестовый стенд из цели узла: фиксированный набор сцен, запуск без графики из командной строки (headless), детерминированные сиды (один сид → точно тот же прогон), машиночитаемая телеметрия. Агенто-управляемый: кодинг-агенты сами гоняют прогоны и сами читают результаты. По аудиту это четырёхфункциональный актив: QA + баланс + источник клипов + формат «AI plays our game». |
| **MTS-сцены** | Три паттерна сцен из старого функционального спека ядра: цепочка комнат / вентиляция / вертикальная шахта. Готовый набор представительных сцен для стенда. |
| **DA / PGG** | Dungeon Architect и Procedural Generation Grid — два покупных генератора уровней. Цель узла: ядро ест уровни через ЗАМЕНЯЕМЫЙ адаптер — DA как минимум, PGG в идеале. Жёстко прибивать ядро к генератору нельзя (правило старого аудита A1). |
| **render proof slice** | Таймбокс ≤2 недель из цели узла (критерий 6): проверить гипотезу стилизации газа и принять решение по рендер-пайплайну за швом. Гипотеза по умолчанию — «квантованная волюметрика» (рецепт Lethal Company: настоящий объёмный туман на низком внутреннем разрешении + постеризация; артефакты дешёвого рендера превращаются в стиль, и он же прячет соло-качество ассетов). С замером миллисекунд кадра на классе минимального железа. |
| **scripted breach** | Запрограммированное событие «прорвало стену»: посреди сессии меняется граф комнат (две комнаты сливаются / открывается новый портал), газ начинает течь по-новому. Это НЕ физическая разрушаемость — её в ставке нет; критерий узла требует «хотя бы одно destruction-driven изменение топологии», и скриптованный прорыв его закрывает честно: трудная часть (сим и сеть переживают смену топологии) — настоящая. |
| **min-spec** | Целевое железо игрока: видеокарта 60-класса с 8 ГБ VRAM (RTX 3060/4060 — самые массовые по Steam Survey) + 16 ГБ RAM. Все замеры производительности — против этого профиля, не против твоей 4090. Валидация дёшево: зажатый по клокам/лимиту мощности профиль на 4090, опционально б/у 3060 за ~$200 ближе к EA. |
| **seam (шов)** | Интерфейс, за которым реализацию можно заменить, не трогая остальное. Швы — главный инструмент против «всё переплелось»: рендер за швом, генератор за швом, T2 за швом. |
| **moat** | Защитный ров: то, что конкурентам трудно повторить. Здесь: первый коммерческий co-op expedition на мелкозернистой мульти-газовой симуляции с разрушаемой топологией (SS13/SS14 — 2D наследие и посевная аудитория, не опровержение). |
| **G3, G4, G6…** | Жёсткие гейты твоей системы: G3 — аппетит не продлевается; G4 — ставка без done_when и kill_by недействительна; G6 — шейп без режущего списка, вердикта по линзам и задачи на рисковейшее допущение недействителен. |
| **kill_by** | Условие убийства ставки: метрика + порог + дата. Дошли до даты или пробили порог — автоматически собирается review, ставка умирает или пересобирается. Назначается на шаге 8, после выбора подхода. |

## 3. Подход A — «Сеть первой: greenfield на фундаменте G4A» (строим)

**Идея.** Самый страшный риск всего направления — «сетевая синхронизация газа не взлетит». Значит, он умирает первым. Первые ~2 недели вообще не трогаем «настоящий» сим: берём НАРОЧНО примитивное поле (тривиальная диффузия на графе комнат — простейшее «газ перетекает из полной комнаты в пустую») и гоним его по сети: NfE listen-server, хост + 2 клиента, кастомный chunked-delta поток, прогоны на стенде без графики, сверка состояния клиентов с хостом по хэшам. Когда контракт потока зафиксирован «в железе» — за ним растёт настоящий сим: T1 мульти-газ с двухуровневыми декларативными типами, **плюс минимальный T2** (внутрикомнатная деталь — для зрелищного клипа и доказательства moat'а), адаптер DA, scripted breach, render proof slice.

**На что ставит.** Расписание съест сетевая модель, а не богатство симуляции; новый минимальный сим за зафиксированным сетевым контрактом получится быстрее, чем любой порт старого. Старый код — только референс.

**Что убивает.** Если chunked-delta не держит консистентность двух клиентов уже на примитивном поле (~2 недели в ставке) — таймбокс-спайк альтернативной модели (1–2 недели, по P6) или смерть ставки.

**Слабое место.** Минимальный T2 сидит в плане с первого дня — а это самый большой кусок скоупа, и его необходимость никем не доказана: может оказаться, что для зрелищного клипа хватает T1 + умного рендера.

## 4. Подход B — «Порт сима из TheLastExit, сеть второй» (переиспользуем)

**Идея.** В старом репозитории есть зрелое одиночное поведение газа. Портируем названный стартовый набор (Gas.Config/Foundation/Topology/T1/T2 + грид-подложка) в GasCoopGame, добиваемся зрелого поведения в стенде, потом накрываем сетью.

**На что ставит.** Зрелость поведения старого сима перевешивает его архитектурный долг.

**Почему я его не рекомендую — это вердикт твоих же аудитов, не мой вкус:**

1. Аудит A1 прямо запретил: прямой перенос кода, использование legacy room id как идентичности, перетаскивание монолита.
2. Гайдрейл multiplayer-readiness требует швов (command/commit, version/change-set, snapshot/delta), которых в старом коде нет — их пришлось бы врезать в чужую архитектуру, это дороже, чем написать минимальный сим заново.
3. Самое тяжёлое: сетевой риск уезжает в КОНЕЦ аппетита — сначала месяц портируем сим, потом узнаём, дружит ли он с сетью. Это анти-паттерн: Stationeers потеряли целый цикл на смене одного только транспорта (откат 09.01.2026 → перевыпуск 28.01.2026). У нас 6 недель, такой удар в конце не пережить.
4. FishNet-бриф той эпохи устарел: P6 зафиксировал «NfE по умолчанию, модель закрыта, вендор заменяем».

B остаётся в протоколе как честно взвешенный и отклонённый. Если ты с этим не согласен — скажи, обсудим, выбор твой.

## 5. Подход C — «Тонкий срез: строится только то, что меряет done_when» (делаем работу ненужной)

**Идея.** Выкинуть всё, что не требуется буквой done_when узла. Только T1 на графе комнат (контейнеры + порталы). T2 — целиком за швом, в этой ставке его НЕТ. Два мета-газа + третий газ чистым конфигом (ровно то, что требует критерий приёмки). Прорыв — scripted событие, никакой системы разрушений. Только DA (PGG отрезан). Сеть — тот же NfE chunked-delta, но на комнатной грануляции (поле крошечное — комнаты, а не ячейки). Render slice по критерию 6.

**На что ставит.** «Представительность» и зрелищный клип (критерий 8) достижимы на комнатном уровне детализации — рендер интерполирует комнатные массы в красивые объёмные потоки, и внутрикомнатная сетка не нужна вовсе.

**Слабое место.** Если render slice покажет КАШУ вместо потоков (газ выглядит как равномерный суп в коробке комнаты, переходы между комнатами — ступеньки), минимальный T2 возвращается в скоуп **посреди аппетита, в незапланированный момент**. C не назначает, когда этот риск проверяется — «когда дойдёт по критерию 6» может оказаться 5-й неделей, и тогда возврат T2 разносит конец ставки.

## 6. Рекомендация: гибрид A+C — «сеть первой, срез тоньше»

**Идея одним абзацем.** Берём из A порядок работ (сетевой риск умирает первым, недели 1–2 — замок репликации над примитивным полем). Берём из C размер скоупа (ничего, что не меряет done_when: T1-only, scripted breach, DA-минимум, два хендлера + газ-конфигом; T2 за швом, в плане его нет). И добавляем одно заострение, которого не было ни в A, ни в C: **render proof slice вытягивается рано (недели 2–4) и официально назначается воротами решения по T2** — рискованное допущение C проверяется в запланированный момент, а не когда придётся.

**Понедельный скетч (12.06 → 24.07):**

| Недели | Что происходит |
|---|---|
| **1–2** (12.06–25.06) | **Замок репликации.** NfE listen-server, хост + 2 клиента, тривиальная диффузия на графе комнат, кастомный chunked-delta поток. Headless-прогоны с детерминированным сидом: хэш поля на хосте и клиентах сверяется каждый тик, посреди прогона — scripted смена топологии. Контракт потока фиксируется. Это рисковейшее допущение всей ставки — оно тестируется ПЕРВЫМ. Бонус окна: Fable 5 доступен до 22.06 — самые трудные сетевые задачи прогоняются самым сильным агентом. |
| **2–4** (≈19.06–09.07) | **Render proof slice** (таймбокс ≤2 недель внутри окна): интерполяция плотности из комнатных масс → квантованная волюметрика в одной сцене стенда; замер миллисекунд на зажатом min-spec-профиле; решение по пайплайну за швом. Выход: вердикт «потоки или каша» + кандидат на первый зрелищный клип (веха ~10.07, открывает параллельные треки). |
| **Ворота T2** (на выходе slice, не позже ~09.07) | «Потоки» → T2 отрезан до конца ставки, экономия ~1–2 недель. «Каша» → минимальный T2 (только для клип-сцены) возвращается, и за него платит другой скоуп (например, DA-адаптер сжимается до hardcoded-сцен и уходит в parked). Аппетит не двигается в любом случае (G3). |
| **3–5** (26.06–16.07) | **Настоящий T1 мульти-газ** за зафиксированным сетевым контрактом: два мета-газ хендлера + третий газ чистым конфигом (критерий 4), DA-адаптер (минимум), scripted breach как постоянная фича стенда, debug-поверхности (масса/потоки/тик/ревизия топологии — критерий 3). |
| **6** (17.07–24.07) | **Добивание критериев:** чистая одиночная сессия + число игроков как конфиг (критерий 7), min-spec замеры (критерий 1), воспроизводимый билд + видео (критерий 3), клип — если ещё не снят. |

**Что в ставку НЕ входит (явный список):**

- физическая разрушаемость (прорыв — скриптованное событие);
- T2 / внутрикомнатная деталь (за швом; возвращается только по вердикту «каша», за счёт другого скоупа);
- PGG, второй генератор (шов адаптера доказывается на DA);
- матчмейкинг и лобби-браузер (по P7 — planned-for-EA, ставку не гейтит; в стенде — прямое подключение);
- save/load (сессии стенда эфемерны, воспроизводимость дают сиды);
- продуктовые визуалы сверх slice — полная визуальная identity газов это узел g-7e15;
- игровой контент: экспедиционный луп, инструменты игрока, UI сверх debug, звук;
- GPU-авторитарная симуляция (запрещено P6: GPU только для визуала), выделенные серверы, >4 игроков, боты.

**Почему гибрид лучше A.** Минимальный T2 — самый большой кусок скоупа A с самым слабым доказательством необходимости. Slice всё равно обязателен (критерий 6) — пусть он и решает, нужен ли T2. Не нужен — сэкономлены ~1–2 недели на самом перегруженном отрезке. Нужен — узнаём к ~09.07, а не на 5-й неделе.

**Почему гибрид лучше C.** Тот же скоуп, но главный риск C (каша) получает запланированную дату проверки и заранее названную цену реакции. C оставляет его сюрпризом.

**Почему не B.** Три удара собственных аудитов (§4): запрет прямого переноса, отсутствие мультиплеерных швов, сетевой риск в конце аппетита.

**Честные остающиеся риски гибрида:**

1. **NfE/DOTS трение** с GameObject-экосистемой (DA — GameObject-мир). Запасной выход заранее санкционирован P6: смена вендора (NGO/FishNet) внутри той же модели, без пересборки дерева.
2. **Производственная скорость не аудирована** — 49-агентный аудит сам честно записал (§7.2): весь осенний календарь стоит на недоказанном предположении о скорости именно этого узла. Лечится только ранним фактом: первые задачи покажут реальный темп за дни.
3. **Категория «networked gas clip» пуста** — зрелищность сетевого газ-сима не доказана никем в природе (аудит §7.4). Незанятая категория = и возможность, и недоказанность. Ранний slice — самая дешёвая проверка.
4. **Min-spec через прокси** (зажатая 4090) может врать. Спот-чек на б/у 3060 (~$200 из бюджета $3k) остаётся опцией ближе к EA, ставку не гейтит.

## 7. Про «инкубацию» — почему я предлагал подождать ночь

Правило из твоего же плея (`os/plays/shape.md`, Notes): когда аппетит больше недели, шейп режется на две сессии — первая раскладывает подходы, выбор делается во второй, **не раньше чем через ночь**. Смысл: сессия, которая сама родила аргументы, не должна сама же и выбирать — свежий текст всегда кажется убедительным; ночь — самый дешёвый сброс этого якоря. s-shape-001 закрыт сегодня в 13:27, эта сессия открыта в 13:39 — ночи не было, поэтому я обязан был показать конфликт. Решение твоё: твоё живое слово выше плея — скажешь «выбираю сейчас», это законный waiver, фиксирую дословно и едем.

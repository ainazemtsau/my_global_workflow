## Correction

The previous recommendation was wrong in its operational reading.

`Bubble-vs-mission constrained hybrid` sounds like building two mechanics. That is not acceptable for the first proof. The first proof must not ask the owner/team to spend time on two different objective systems.

Correct route:

**Build/evaluate one first proof only: Bubble-first.**

The proof loop is:

1. find sleeping gas;
2. wrap/bubble a bounded gas state;
3. carry visible live custody through a hostile route;
4. avoid rupture;
5. if rupture happens, released gas re-enters the world simulation;
6. decide whether to return for another bubble.

Mission-value-first remains only a paper sanity-check and fallback, not a parallel playable branch:

- paper sanity-check: does Bubble make gas itself feel valuable enough?
- fallback: if owner rejects membrane/custody/route-carry scope, return to mission-value-first.

## Updated recommendation

Recommendation changes from "Bubble-informed hybrid first" to:

**Bubble-first as the single next proof, with no second mechanics lane.**

Bad alternatives:

- Generic Bottle/canister-first is bad because it hides the gas and becomes container stats.
- Mission-value-first now is bad because it spends first-proof energy away from the strongest hook.
- Hybrid comparison is bad because even if phrased as constrained, it can become two prototypes and waste time.

## What this does not freeze

This still does not canonize:

- exact membrane ring implementation;
- final bubble physics;
- exact gas roster, names, tools, containers, cargo economy, timers, progression;
- creature AI;
- magical sleep/wake;
- `ОНО ДЫШИТ` title ontology.

## Owner decision

The owner decision is now narrower:

Approve Bubble-first as the single first proof, or reject Bubble-first and fall back to mission-value-first.

# Газовый движок — сводная картина реворка (pre-R#)

**Дата:** 2026-06-26 · **Статус:** ПРЕДЛОЖЕНИЕ / синтез для ревью владельцем → основа для `R#`.
**НЕ применено к состоянию.** Перератификация Факта 2, суперсед `c-exec-013`, смена контракта — отдельным RESULT после «да».
Все ссылки даны от корня направления `live/indie-game-development/` (SPEC = `knowledge/g9c41-gas-engine-SPEC.md`).

---

## 0. TL;DR

Вся сегодняшняя трения — дверь, спор про метки, путаница 50/25см, вопрос про слои — это **симптомы ОДНОГО корня**: при консолидации (~11 доков → один SPEC) **owner-confirmed СЛОЙНУЮ архитектуру координирующего grid'а сплющили в газо-центричную «одну 50см-сетку»**, и build S0 унаследовал сплющенную версию. Починка — одна реставрация: поднять слойный grid в ведущий кадр; **газ / структура / размещение = первоклассные слои, каждый на своём разрешении**; метки = авторинг слоя структуры; инвертировать инвариант «геометрия не тоньше газа» детерминированной целочисленной проекцией. Дверь и разрушаемость становятся решаемыми оба.

## 1. Корень: каркас есть, но он не ведёт

Слойная архитектура **в живом каноне присутствует**, но спрятана:

- **Механизм — есть** (SPEC §9 шов 7, `:97`): «Межслойное чтение — ТОЛЬКО против ЗАФИКСИРОВАННОЙ (committed) ревизии в grid-координатах (publish-events + pull read-model); слой НИКОГДА не читает чужое live mid-tick состояние». Плюс: чексумма «по типу/**слою**» (§9.5 `:95`), запас адресов «под доп. реплицируемые **слои**» (§9.10 `:100`), `collapse/expand` «по-слойно» (§9.4 `:94`).
- **Крисп-формулировка — есть, но не в SPEC**: «GRID = shared coordinate/address space + cell→region ownership map + event bus + revision/commit clock — NOT a data router/god-object; each layer publishes events + read-model» живёт в `NOW.md` (≈`:755-758`) и в **архивном** `work/archive/converge-g-9c41.md` (frozen-legacy, не авторитет).
- **Подача в SPEC — газо-центрична**: открывается Фактом 2 (`:24-29`) «**ОДНА** клеточная модель 50см изотропно»; слойный каркас — мелким шрифтом в §9 «швы» (форма данных, поведение-позже).

**Следствие:** build, читающий SPEC сверху, видит «одна 50см газовая сетка», выводит из неё ВСЁ (топология, проёмы, размещение) на 50см → газо-центричный дрейф. Отсюда все четыре симптома.

## 3. Целевая архитектура (ведущий кадр)

**GRID = координирующий каркас**, НЕ god-object:
- общее координатное/адресное пространство;
- карта владения `cell → region/sector`;
- шина событий (publish-events);
- такт-ревизия (commit clock).

**СЛОИ** — каждый со своим разрешением и разметкой; общаются ТОЛЬКО через committed-события + read-model (лаг 1 тик, никогда live mid-tick — same-tick кросс-слой = рассинхрон, SPEC §9.7):

| Слой | Разрешение | Роль | Статус |
|---|---|---|---|
| **ГАЗ** | 50см, объём | авторитетный сим (поток, масса) | **без изменений** — заратифицированный выигрыш цены |
| **СТРУКТУРА** | 25см, грани | единственный источник структурной правды: стены/полы/проёмы/пробиваемость; **авторится метками** | новый первоклассный слой |
| **РАЗМЕЩЕНИЕ** | 25см, точки | объекты/предметы | **разрешение зарезервировать, механику отложить** (объектов пока нет) |

**СВЯЗЬ (ключевое):** структура (25см) проецируется **ВНИЗ** в газ (50см) как **детерминированная целочисленная проводимость**: «сколько 25см-подграней открыто / 4 → проводимость грани (0–4 из 4)». Односторонняя, статическая, целочисленная →
- детерминизм сохранён;
- **газ остаётся одной 50см-сеткой** (нет газ↔газ re-flux шва — того самого, что лок 50см и убрал; убран остаётся);
- «25см-дыра = ¼ проводимости = ¼ протечки» — механики работают «как будто честные 25см», а газ дёшев.

**АВТОРИНГ структуры/размещения:** разметка на **переиспользуемых модулях** (ставит владелец ИЛИ ассистент через MCP) по **независимым правилам**; геометрические метки **валидируются против геометрии** (метка ⟂ мешу = RED); чисто семантические метки (пробиваемость/материал) = дизайн-данные; **билдер читает, но не выдумывает их, чтобы пройти тест** (анти-чит c-exec-012 сохранён — через валидацию, не через запрет авторинга).

> Визуальная схема разрешений (50см газ ↔ 25см геометрия ↔ ¼-проводимость) — в чате этой сессии (`resolution_split_gas50_geometry25`).

## 4. Что меняется (объём `R#`)

1. **Поднять слойный координирующий grid в ВЕДУЩИЙ кадр SPEC.** Факт 2 переформулировать: газ = 50см; grid = координирующий слой; структура/размещение = слои. (Реставрация того, что было сплющено — не новое изобретение.)
2. **Перератифицировать лок размера клетки как СПЛИТ:** `gas_res=50` (сохранён) + `geometry_res=25` + `placement_res` (зарезервировать). **Инвертировать инвариант** `resolution_tag`: геометрия МОЖЕТ быть тоньше газа, связь = детерминированная целочисленная проекция (определить её; она снимает ровно тот алиасинг, которого боялся старый инвариант). Заменяет `c-exec-013-call.md:119-124`.
3. **Переформулировать анти-подмену / генератор-слепоту:** различить метаданные-на-модуле (легитимно; билдер-через-MCP ОК под независимыми правилами + геом-валидация) от разметки-уровня билдером (по-прежнему запрещено).
4. **Правило двери:** проём = объявленная апертура слоя структуры (25см), валидируется против геометрии; газ читает спроецированную проводимость. Заменяет «lateral = full shared boundary» + переписать RED RV2/RF1 (силами независимого test-author).
5. **Суперсед `c-exec-013`** реворкнутым S0, построенным под слойный кадр.
6. **Пролом/разрушение:** per-face поля заполняются из меток слоя структуры; базовый пролом (S5) = открыть латентные подграни → смена спроецированной проводимости → поток газа. 25см-дыры → ¼-проводимость. Полная разрушаемость остаётся **parked**.

## 5. Что СОХРАНЯЕТСЯ (не перетряхиваем)

- Детерминизм по построению (целые, без float в авторитетном пути).
- **Газ на 50см** (выигрыш цены; газ — одна сетка; re-flux шва нет).
- Шов R13/R14 (ядро = чистая C#-либа; Unity = адаптеры).
- Анти-чит билдера (нельзя подделать, чтобы пройти тест — теперь через валидацию).
- Owner-eye сплит «точно/весело».
- Вся model-agnostic работа S0 (закон грань-потока, скан детерминизма, песочница, грубый сим).

## Глоссарий (для владельца)

- **Слой (layer)** — отдельная подсистема (газ / структура / размещение), держит свои данные, со своим разрешением.
- **Координирующий grid** — общая «адресная книга» пространства: где что, кто чем владеет, через него идут события и перевод координат. Данные НЕ хранит.
- **read-model** — «вид только для чтения» состояния слоя на зафиксированной ревизии (как read-replica БД). Слои читают друг друга только так.
- **committed-ревизия / такт** — слой публикует состояние «на момент N»; другие читают этот зафиксированный снимок (лаг 1 тик), не живое.
- **Проекция** — перевод тонкой структуры (25см) в грубую газовую проводимость (доля открытых подграней).
- **Проводимость грани** — насколько грань пропускает газ (0 = стена, 4/4 = полностью открыто, 1/4 = 25см-дыра).
- **region_id** — метка-раздел (какой комнате принадлежит клетка), есть на всех тирах.
- **resolution_tag** — пара (geometry_res, gas_res): разрешение геометрии и газа.
- **Вокселайзер** — читатель уровня: геометрия → целочисленная сетка клеток с гранями.
- **Генератор-слепой** — читает то, что РЕАЛЬНО построил генератор DA, а не ручную правку конкретного уровня.
- **Пролом / breach** — открыть закрытую грань → газ потёк (без рантайм-ремеша). S5 = базовый; полная разрушаемость — parked.

core_frame_plain_language: |
  "ОНО ДЫШИТ" is a co-op extraction/salvage game about entering a gas-dangerous underground city and bringing out value
  through air that behaves like a hidden physical antagonist.

  At the start, the gas is often not clearly visible as a full cloud. Players read indirect signs and instruments:
  frost, residue, dust movement, ventilation breathing, pressure/temperature/density anomalies, corrosion, muffled sound,
  strange route behavior, local damage, partial instrument spikes.

  The team builds a working picture:
    where gas is, how much space it occupies, what layer/height it sits in, what state it is in, what may trigger it, and
    what it will do to route/player/value if disturbed.

  Then the team acts physically:
    open/close, vent, break, redirect, heat/cool, isolate, use/avoid condition, stabilize, release, rescue, or later
    possibly capture a gas state.

  The mission is won when people and mission value reach safe return or a useful proof/procedure is preserved.
  Gas extraction itself is only one candidate form of mission value, not the default law.

core_loop_player_action_altitude: |
  1. Contract / reason to enter:
       Team has one plain mission: retrieve value, rescue, recover data/object, shut down/purge, prove a procedure, or later
       capture a gas state.

  2. Quiet / hidden atmosphere:
       Floor is not immediately "colored cloud everywhere".
       Gas presence is often indirect: traces, symptoms, pressure, dust, frost, instruments, environment oddities.

  3. Detection-first read:
       Players estimate:
         - where gas is;
         - what layer/height/area it occupies;
         - how much there may be;
         - what state it is in;
         - what may trigger or reveal it;
         - where route/value/player risk sits.

  4. Small confirmation / bounded test:
       Players do not solve the whole floor at the door.
       They move, probe, open, listen, measure, watch secondary effects, and learn through consequences.

  5. Physical intervention:
       Players alter gas/building/condition:
         doors, vents, route, pressure, flow, cold/hot pockets, breaking topology, isolating rooms, stabilizing, releasing,
         quenching, or rerouting.

  6. Manifest consequence:
       Gas becomes more visible, active, dangerous, useful, route-changing, player-affecting, or value-affecting.
       Consequence must be readable and attributable.

  7. Mission value / stake:
       The stake may be:
         - object/data/person/proof;
         - route confidence/procedure;
         - shutdown/salvage result;
         - ordinary cargo/object;
         - captured gas state later, if that branch is chosen.
       Stake is not banked until safe return or accepted mission boundary.

  8. Return liability:
       The team owes the return route:
         worse air, worse route, occupied carrier, spent tools, unstable value, injured/downed player, triggered gas,
         blocked passage, or deeper commitment.

  9. One-more-pass pressure:
       The floor should tempt:
         "one more room",
         "one more passage",
         "one more object",
         "one more signal",
         "one more level down",
         "one more run after base prep".
       This temptation must be rationally risky, not just content grind.

  10. Handoff:
       continue inside floor / go deeper / return-bank attempt / retreat / release-abandon / partial disaster /
       failure seed / knowledge-readiness handoff.

one_more_pass_principle:
  accepted_as: "design principle / map invariant, not exact mechanic"
  plain: |
    The game should deliberately create a "one more turn" pull.

    In this game the phrase becomes:
      one more passage,
      one more room,
      one more floor/level,
      one more depth step,
      one more run after base prep.

  why_it_matters: |
    Without this pull, the game may become a correct gas incident simulator:
      enter, solve, leave.

    The desired loop is stronger:
      we saw another signal;
      we almost understand this;
      one more room may pay;
      one more depth step may reveal better value;
      one more base prep makes the next attempt tempting;
      but return liability and gas consequences make the choice scary.

  allowed_sources_of_pull:
    - visible but not fully understood gas trace
    - partially mapped route or alternate passage
    - deeper signal / stronger mission value
    - base-prep unlock that makes a previous failure worth retrying
    - recoverable mistake that invites a cleaner second attempt
    - route knowledge / procedure / marked source
    - "we can do it better now" mastery
    - value spotted beyond current safe margin
    - optional room with clear risk
    - post-run incident report / replay showing an avoidable failure
    - contract chain that points one step deeper

  forbidden_forms:
    - grind treadmill
    - hidden retention manipulation
    - daily quest / checklist as substitute for systemic pull
    - universal hard timer forcing hurry by default
    - "one more" only because numbers go up
    - base crafting that replaces field value
    - content quantity instead of decision pressure
    - FOMO or live-service framing

  base_global_connection: |
    Base/global gameplay should convert run output into the next temptation:
      - better read;
      - better preparation;
      - route confidence;
      - marked source;
      - known failure cause;
      - partial readiness;
      - a tool/insurance/procedure that invites one more attempt;
      - a deeper contract now barely within reach.

    Base should not be a spreadsheet economy center. It should make the player think:
      "Now we understand enough to try one more room / one more level / one more run."

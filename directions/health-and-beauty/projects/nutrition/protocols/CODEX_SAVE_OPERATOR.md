# Codex Save Operator Protocol

```yaml
protocol:
  schema: codex_save_operator_protocol.v1
  project: "Питание"
  status: active
  purpose: apply_approved_nutrition_state_update_packets
```

## Role

Codex Save Operator applies user-approved `nutrition_state_update_packet.v1` packets to repository files and returns evidence.

Codex Save Operator does not give nutrition advice, does not interpret nutrition content, and does not decide what the user's nutrition state should be.

## Required Input

Codex may save only when the user provides an approved packet:

```yaml
nutrition_state_update_packet:
  schema: nutrition_state_update_packet.v1
  packet_status: approved_by_user
  external_write_claimed: false
  target_files: []
  updates: {}
```

If `packet_status` is not `approved_by_user`, Codex must refuse to write and ask for explicit approval.

## Allowed Write Targets

Codex may edit only these durable state files:

- `directions/health-and-beauty/projects/nutrition/project_files/01_NUTRITION_BASE.md`
- `directions/health-and-beauty/projects/nutrition/project_files/02_MENU_PREFERENCES.md`
- `directions/health-and-beauty/projects/nutrition/project_files/03_ACTIVE_CYCLE.md`
- `directions/health-and-beauty/projects/nutrition/project_files/04_TRACKING_AND_EXCEPTIONS.md`
- `directions/health-and-beauty/projects/nutrition/project_files/05_REVIEW_AND_SYNC.md`

Codex may update package docs only when the user explicitly asks for package maintenance. Normal state saving must not edit README, manifest, instructions, or dry-run protocol.

## Refusal Triggers

Return `NEEDS_INPUT` without writing if:

- the packet is absent;
- the packet is not approved by the user;
- the packet targets files outside the allowed write targets;
- the packet asks Codex to decide nutrition content;
- the requested change requires medical, clinical, supplement, fasting, lab, training, cardio, or recovery advice;
- the packet conflicts with the current repository files;
- the target file has unexpected structure that prevents a safe small edit.

## Save Procedure

1. Read the approved packet.
2. Verify schema, approval status, target files, and allowed path boundary.
3. Read current target files.
4. Apply the smallest repository edit that records the approved state.
5. Preserve end-of-file markers as the final non-whitespace content.
6. Run validation:
   - file existence;
   - target path boundary;
   - read-back of changed anchors;
   - diff check;
   - no nutrition advice generated by Codex.
7. Return changed files, diff summary, read-back anchors, validation results, and commit/push status if requested.

## Output Shape

```yaml
codex_save_result:
  return_state: DONE | NEEDS_INPUT | STUCK
  packet_schema: nutrition_state_update_packet.v1
  approved_by_user: true | false
  changed_files: []
  forbidden_path_check:
  read_back_evidence: []
  validation_results: []
  nutrition_advice_given_by_codex: false
  unresolved_issues: []
```

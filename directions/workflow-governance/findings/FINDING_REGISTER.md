# FINDING_REGISTER

This is the canonical Workflow Governance finding register.

## Finding schema

```yaml
finding:
  finding_id:
  status:
  area:
  severity:
  confidence:
  evidence:
  recommendation:
  alternatives:
  patch_needed:
  approval_needed:
  next_action:
```

## Status values

- open
- accepted
- rejected
- deferred
- resolved
- needs_user_decision
- needs_research

## Severity values

- P0
- P1
- P2
- P3
- nonblocking

## Rules

- Evidence must cite GitHub paths or source-backed research references.
- Separate source-backed facts vs inference.
- Do not create a repository patch from a finding until approval.

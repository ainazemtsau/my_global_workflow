## Purpose / boundary

This note defines the locked experience model for the selected primary bet:

Expedition / session-structured living reactive gas sandbox

It is not:

- a full game design document;
- a proof question;
- a playable proof;
- a prototype backlog;
- a tech plan;
- a validation plan;
- a metrics plan;
- a mission catalog;
- a progression system;
- a concrete reaction recipe set;
- a full lore document.

The stable product identity is:

A 2–4 player co-op expedition game where the team chooses a dangerous procedural gas environment, enters it to extract value, manipulates gas and topology to create temporary safety, handles unstable carried loot, and decides whether to push deeper, reroute, retreat or abandon the run.

## Core loop

Choose expedition target / level / depth → enter dangerous procedural gas space → read gas and topology → create temporary safe access → find or expose valuable material → carry / secure unstable value → manage gas, route and cargo consequences → decide whether to push deeper, reroute, retreat or abandon → resolve extraction, partial extraction, loss or retreat.

The important addition is that extracted value is not always inert. Some loot may be gas-bearing, unstable, reactive or fragile. The team is not only trying to reach value; it is trying to bring dangerous value back without turning it into a new gas event.

The loop depends on these pressures:

- The environment is already alive.
- Gas exists, moves, accumulates, reacts, blocks access and changes outside direct player presence.
- Safety is temporary.
- Extraction creates commitment.
- The return path matters.
- Carried value can become danger.

## Tension source

The primary tension source is expedition commitment under living environmental and cargo consequence.

The team wants to extract more value, reach a better target or push into a deeper route, but every intervention can make the environment harder to read, harder to cross or harder to escape.

Pressure should not be only a race timer. The tension comes from a combination of:

- exposure to gas;
- route degradation;
- gas migration;
- reaction risk;
- unstable carried loot;
- limited attention between teammates;
- greed for more value;
- uncertainty about the return path;
- fear that one mistake can create a readable chain consequence.

The intended player questions are:

- Can we make this route safe enough for long enough?
- Is the deeper target worth the return-path risk?
- Can we carry this unstable value safely?
- Should we secure the loot now or move before gas shifts?
- What happens if this container breaks?
- Do we still understand what the gas is doing?
- Do we push, reroute, retreat or abandon?

This prevents the game from becoming a loose “cool gas sandbox.” The run has value pressure, survival pressure, cargo pressure and return-path pressure.

## Session / run structure

Session/run structure:

bounded co-op expedition run with pre-run target choice and in-run push / reroute / retreat / abandon decisions.

The session is not one linear handcrafted path.

The run structure is:

1. Target choice  
  The team chooses a level, depth, expedition target, route or danger band.

2. Ingress  
  The team enters a generated gas-dominated environment with incomplete knowledge.

3. Reading phase  
  Players observe gas behavior, room layout, doors, corridors, openings, vertical shafts, possible safe pockets and route risks.

4. Intervention phase  
  Players manipulate topology and gas conditions to create temporary access or safer timing.

5. Extraction / cargo phase  
  Players reach value, handle it, carry it, secure it or decide whether it is too dangerous to take.

6. Consequence phase  
  Gas movement, reactions, off-player simulation, unstable loot and team mistakes change the state of the environment.

7. Decision point  
  The team chooses whether to push deeper, reroute, secure current value, retreat, abandon the level or attempt another target.

8. Run resolution  
  The run ends in extraction, partial extraction, retreat, loss, abandonment or a changed environment that may matter later if persistence is supported.

Bad procedural outcomes should not automatically hard-stop the game. A generated space can be too risky, too expensive, low-value or poorly suited to the current team. That is acceptable if players can read enough of the danger and choose retreat, reroute or another expedition.

## Minimum action contour

Choose expedition risk → enter → read gas/topology/value → alter route or flow through spatial decisions → create temporary safety or access → recover unstable value → coordinate carrying or securing → manage readable gas/cargo consequence → push / reroute / secure / retreat / abandon.

This is a shape-level contour, not a tool roster.

The minimum experience requires that players can:

- choose an expedition target, level, depth or route;
- enter a gas-threatened procedural space;
- read danger at room, route and topology scale;
- change access or flow through spatial decisions;
- create temporary safe pockets, timings or routes;
- find, expose, collect or recover something valuable;
- carry value through a dangerous changing environment;
- deal with unstable gas-bearing loot that can fall, break, leak or release gas;
- coordinate with teammates to secure valuable cargo;
- suffer readable consequences from gas, topology, reactions or cargo handling;
- decide whether to push deeper, reroute, secure, retreat or abandon.

Not committed at this level:

- specific tools;
- equipment list;
- inventory slots;
- backpack, suit or rig rules;
- UI;
- player classes;
- mission templates;
- reaction recipes;
- cargo formulae;
- exact rupture, explosion or knockback rules.

A key co-op handling pattern:

A player may carry unstable value in hand, but securing it into a safer carry state may require teammate interaction.

This is not a locked inventory system. The exact equipment, slots, UI and rules are not defined here. The commitment is only that valuable cargo can require cooperation to stabilize and transport.

## Role of gas as core of experience

Gas is the core experiential medium.

Gas is:

- threat;
- tool;
- route-shaper;
- consequence carrier;
- cargo risk;
- co-op danger source;
- run story generator.

Gas is not decorative VFX and not only area damage. The team is playing with and against a living environmental medium.

Gas should be able to:

- block routes;
- fill or empty rooms;
- move through doors, corridors, openings and shafts;
- create temporary safe or unsafe spaces;
- persist outside player presence;
- interact with topology;
- interact with other gas states;
- appear inside unstable carried loot;
- create delayed consequences that players can understand later.

The run should be remembered through gas-state stories:

- We opened a route and poisoned our return path.
- We made the lower corridor safe, but only temporarily.
- We carried unstable loot through a reactive gas pocket and almost lost it.
- Someone dropped a container, it broke, released gas and changed the whole room.
- We abandoned the deeper target because the gas pattern made escape too risky.

Gas may feel living, semi-living, mutating or reactive, but it must remain readable enough for agency. Complexity is acceptable; opaque chaos is not.

## Role of gas types and reactions

Reaction grammar status:

shape-level truth

Concrete reaction recipes status:

hypothesis material, not hard commitments

Gas types and reactions are part of the selected bet because they make the environment feel alive and worth expedition-style play. However, this model does not lock specific substances, tiers, recipes, balancing, reaction tables, formulas, rupture rules, explosion rules or knockback rules.

At experience level, gas types should differ through readable traits such as:

- density;
- persistence;
- toxicity;
- visibility;
- volatility;
- mutability;
- tendency to block or open routes;
- reaction risk;
- usefulness for temporary safety;
- danger when released from carried loot.

Gas may also break down, mutate or release another gas state under certain conditions. At this level, this means only that gas behavior can transform into another readable state. It does not mean a locked recipe like “gas A always becomes gas B.”

Reactions exist to serve these experience roles:

- Readable cascade;
- Risk amplifier;
- Route / hazard transformer;
- Cargo-mediated event;
- Co-op danger source;
- Living-world signal.

Possible consequence classes include:

- hazard transformation;
- route contamination;
- loss of temporary safety;
- pressure event;
- explosion-like force;
- knockback;
- chain loss of carried loot.

These are example-level consequence classes, not locked recipes. They do not define required substances, required pairings, exact formulas, frequencies, damage numbers, rupture math, force rules or production balancing.

The target is readable systemic transformation, not recipe memorization and not random chain explosions.

## Role of topology / grid

Topology/grid is the causal surface through which players understand and affect gas.

Rooms, corridors, doors, openings, vertical shafts, pits, high pockets and portals are not decoration. They are the structure through which gas moves, gets blocked, accumulates, escapes, reacts and threatens the team.

In this bet, topology creates expedition decisions:

- which route to open;
- which connection to keep closed;
- which room can become temporary safety;
- which vertical space is useful or dangerous;
- where unstable loot can be handled safely;
- where gas might move while players are away;
- how one player’s local action affects another player’s route;
- whether the return path is still viable.

The player does not need to think in technical terms like “simulation grid.” The player needs to understand practical spatial consequences:

- this door;
- this room;
- this shaft;
- this pocket;
- this route;
- this return path;
- this gas-filled cargo risk.

Readability at room and route scale is mandatory. If players cannot form a practical mental model of how gas and topology interact, the bet becomes chaos instead of mastery.

## Role of procedural generation

Procedural generation player-facing function:

unknown living route pressure

Procgen is not included because it is generally good, cheaper to produce or merely replayable. Its player-facing function is to make each expedition a risk assessment problem around unknown topology, gas state, value location, unstable cargo and return-path viability.

Procgen should give the player:

- a reason to choose between levels, depths, routes or targets;
- uncertainty about where danger, value and safe pockets are located;
- uneven but readable risk/reward spaces;
- different gas/topology situations across runs;
- different cargo-handling problems across runs;
- different return-path risks across runs;
- situations where retreat, reroute or abandonment are valid decisions;
- environments that cannot be solved as fixed authored routes.

For Expedition, procedural unevenness is acceptable if the player can read and respond to it. A generated level can be unfavorable, too risky, too deep, low-value or not worth the attempt. That is not automatically failure. It becomes part of the expedition judgment.

Procgen should be felt as:

We chose this expedition, read this generated space, made a route bet, found unstable value, changed the environment, and now must live with the consequences.

## Co-op pattern

Co-op pattern:

shared environmental and cargo improvisation under route commitment.

Players are not just doing parallel solo tasks. They share one gas/topology/cargo state.

One player’s action can:

- make another route safer;
- contaminate another route;
- create or remove temporary safety;
- release gas from carried loot;
- cause another player to drop cargo;
- force the team to change extraction plans.

The co-op pattern should create:

- Distributed attention;
- Coordinated timing;
- Imperfect information;
- Cargo cooperation;
- Route commitment;
- Recovery moments.

This model does not require fixed classes. Roles may emerge from position, carried value, risk, route state or who is closest to a consequence.

## Harm / friendly-fire pattern

Harm / friendly-fire status:

Core as indirect systemic and cargo-mediated co-op harm.

Severity is an optional intensity axis.

Opaque direct grief damage is a non-goal.

The chosen bet needs harmful co-op because the shared gas/topology/cargo environment is the main source of consequence. If teammates cannot endanger each other through environmental decisions, the co-op loses much of its systemic edge.

However, harm must be readable and attributable. The target is not arbitrary PvP or trolling. The target is:

Your action changed the environment or cargo state in a way that put me at risk, and we can understand why.

Core harm patterns include:

- opening a door lets gas into another player’s route;
- closing or delaying access traps a teammate in worsening gas;
- forcing gas movement clears one area but contaminates another;
- triggering a reaction changes the safety of a room;
- overcommitting to loot makes the return path worse;
- dropping gas-bearing loot releases gas;
- damaged loot reacts with existing gas;
- explosion-like force or knockback causes other players to drop their loot;
- dropped loot breaks and creates a chain consequence.

A permitted high-intensity consequence chain class is:

A teammate mishandles unstable gas-bearing loot → gas escapes → the local gas state changes → nearby players lose safety, route clarity or cargo control → the team must recover, reroute, retreat or abandon.

This is acceptable only if the chain is readable enough that players understand what happened and can improve next time. It is not a locked recipe, not a required event sequence, not a damage formula and not a commitment to physics comedy.

Non-goal:

- hidden causality;
- random teammate punishment;
- direct griefing as the main fun;
- damage that feels detached from gas/topology/cargo consequence;
- physics comedy where chaos replaces mastery;
- opaque PvP damage as core play.

## Player / value promise

A 2–4 player team enters dangerous procedural gas spaces to extract value, survives by reading and manipulating a living environment, handles unstable gas-bearing loot, and remembers runs through route choices, temporary safety, gas reactions, cargo mistakes, overcommitment, retreat and extraction.

The promise is not:

Solve a fixed mission.

The promise is:

Make an expedition judgment inside a living gas/topology/cargo system.

The player should value the game because:

- gas is the face of the experience;
- the environment behaves with persistent consequence;
- topology makes danger understandable and actionable;
- procedural spaces create real route judgment;
- extracted value can itself be dangerous;
- co-op matters because everyone shares one environmental state;
- failure can become a story of retreat, loss, chain reaction or abandonment;
- mastery comes from reading, anticipating and shaping gas/topology behavior.

The strongest session memory should sound like:

We chose a deeper extraction, found unstable gas-bearing value, made a risky route temporarily safe, damaged the return path through our own gas decisions, almost lost the cargo after a teammate dropped it, then chose whether to push for more or escape with what we had.

## Anti-audience

This bet is not for players who primarily want:

- pure combat where the environment is secondary;
- clean authored puzzle solutions with guaranteed solvability;
- low-friction cozy co-op without teammate-caused danger;
- deterministic mission scripting where every run has a designed answer;
- solo-first play where co-op is optional decoration;
- fixed role/class execution more than environmental improvisation;
- progression grind as the main reason to play;
- total clarity at all times with no uncertainty, retreat or loss;
- chaotic physics comedy where causality does not matter;
- hardcore scientific simulation where physical correctness matters more than readable game consequence;
- frictionless loot pickup where extraction cargo is never dangerous;
- co-op without communication, timing or mutual dependence.

The intended audience must tolerate uncertainty, environmental risk, partial information, unstable cargo, imperfect team decisions and the possibility that the correct expedition decision is to leave, reroute or abandon value.
